## Affected Pages/Routes
- `/api/webhooks/github` — HMAC verification, event routing, installation validation
- `/api/pipeline/*` — Pipeline CRUD, stage results, findings queries
- `/pipeline` (dashboard view) — Pipeline list, detail, quality trends

## Key Interactions to Verify
- PR opened on connected repo triggers full pipeline (webhook → clone → 4 stages → comment)
- Force-push to PR triggers new pipeline run, comment updates in-place
- Stage failure results in SKIPPED verdict, other stages complete normally
- Dashboard shows pipeline results with expandable stage details

## Edge Cases
- Invalid HMAC signature on webhook (must reject)
- Path traversal in AI tool call (read_file("../../etc/passwd") must reject)
- All 4 stages fail simultaneously (error comment posted)
- Very large PR diff (>10K lines) — must truncate or handle gracefully
- Clone fails (network, auth, disk) — pipeline marked FAILED
- Claude API rate limit during stage execution — retry with backoff
- Two stages complete at same millisecond — mutex prevents lost update on comment
- Unknown GitHub installation sends webhook — 404

## Critical Paths
- Full pipeline: PR opened → webhook → HMAC verify → clone → parallel stages → DB write → comment posted
- Re-run: force-push → new pipeline_run → stages re-execute → comment re-rendered from latest run
- Cross-repo match: finding embedded → vector search returns precedent from different repo
- Dashboard render: pipeline_runs query → stage_results join → findings expand

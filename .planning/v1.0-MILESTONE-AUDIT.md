## Tech Debt

### Phase 1: Foundation
- App.tsx uses plain `fetch()` instead of Hono RPC client. The RPC client exists and is typed — this was a deliberate choice for proof-of-life. Phase 2+ continued with `fetch()` pattern.

### Phase 4: Search & Intelligence
- Deprecated `searchCaptures` function retained in `packages/api/src/db/queries/search.ts` as a backward-compatibility bridge to `searchUnified`. Dead code — safe to remove.

### Phase 5: Dashboard Enrichments
- CAPT-08 (archived captures remain searchable) is fulfilled implicitly because `PATCH status=archived` does not call `deindexCapture`. This is the correct behavior but should be documented as intentional so a future refactor doesn't accidentally break it by adding `deindexCapture` to the PATCH handler.

### Cross-Phase
- `GEMINI_API_KEY` vs `GOOGLE_GENERATIVE_AI_API_KEY` cosmetic inconsistency in startup and error messages. Both are mapped correctly at runtime — no functional impact.
- Triage modal retains `animate-fade-in`/`animate-scale-in` CSS classes that were removed from command palette. Intentionally kept — triage modal doesn't have the mode-switching flash issue.

### Total: 5 items across 4 areas (none blocking)

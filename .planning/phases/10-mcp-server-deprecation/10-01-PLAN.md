<interfaces>
<!-- MC API endpoints the MCP tools will call. These are the data contracts. -->

GET /api/projects — returns { projects: Array<{ slug, name, host, ... healthScore, riskLevel, copyCount, lastCommitTime, ... }> }
GET /api/projects/:slug — returns { project: { slug, name, host, branch, dirty, dirtyFiles, lastCommitHash, lastCommitMessage, lastCommitTime, ... } }
GET /api/risks — returns { critical: Finding[], warning: Finding[], riskCount: number, summary: string }
GET /api/health-checks — returns { findings: Finding[], total: number } — optional ?severity=critical|warning|info filter
GET /api/health-checks/:slug — returns { findings: Finding[], riskLevel: string }
GET /api/copies — returns { copies: Copy[], total: number }
GET /api/copies/:slug — returns { copies: Copy[], projectSlug: string }
GET /api/sprint-timeline — returns { projects: Array<{ slug, segments, totalCommits }>, focusedProject, windowDays } — optional ?weeks=N

Where Finding = { projectSlug, checkType, severity, detail, metadata, detectedAt, isNew }
Where Copy = { projectSlug, host, remoteUrl, branch, headHash, lastCheckedAt, isStale }

Portfolio-dashboard tool mapping (MIGR-01):
  portfolio_status      -> project_health  -> GET /api/projects + GET /api/health-checks
  project_detail(proj)  -> project_detail  -> GET /api/projects/:slug + /api/health-checks/:slug + /api/copies/:slug
  activity_timeline     -> project_health  -> GET /api/projects (includes lastCommitTime)
  find_uncommitted      -> project_risks   -> GET /api/risks
  sprint_history        -> project_detail  -> GET /api/sprint-timeline?weeks=N
</interfaces>

    project_health:
    - Calls GET /api/projects and GET /api/health-checks
    - Returns formatted text showing each project with healthScore, riskLevel, copyCount
    - Accepts no input parameters (reports on all projects)

    project_risks:
    - Calls GET /api/risks
    - Returns critical and warning findings grouped by severity
    - Accepts optional severity filter ("critical" | "warning")
    - Shows project name, check type, detail, duration for each finding
    - Returns "No active risks" when riskCount is 0

    project_detail:
    - Calls GET /api/projects/:slug, GET /api/health-checks/:slug, GET /api/copies/:slug
    - Returns deep status for one project (branch, dirty state, latest commit, findings, copies)
    - Requires slug input parameter

    sync_status:
    - Calls GET /api/health-checks and GET /api/copies
    - Filters findings to sync-related check types: unpushed_commits, no_remote, broken_tracking, deleted_branch, unpulled_commits, diverged_copies
    - Returns sync report grouped by check type
    - Returns "All projects in sync" when no sync issues found

    All tools:
    - Return { content: [{ type: "text", text: "..." }] } format
    - Catch fetch errors and return error text content (never throw)
  </behavior>

<task type="auto">
  <name>Task 1: Create session startup hook and register MCP server</name>
  <files>~/.claude/hooks/risks-digest.sh</files>
  <action>
    1. Create `~/.claude/hooks/risks-digest.sh`:
       - Bash script calling MC API /api/risks via curl
       - `MC_API="${MC_API_URL:-http://100.x.x.x:3000}"`
       - `RISKS=$(curl -sf --max-time 5 "$MC_API/api/risks" 2>/dev/null) || exit 0`
       - Parse with python3 (same pattern as worklog-digest.sh)
       - Format per locked decisions:
         * If no critical AND no warning: exit 0 (zero noise when healthy)
         * Header: `RISKS ({N} critical, {M} warnings):` — only include present severity types in header
         * Critical risks: listed individually as `  [red circle emoji] {projectSlug}: {detail}`
         * Warnings: summarized as `  + {N} warnings (see dashboard)` — NOT listed individually
       - Output appears AFTER worklog digest (risks come after worklog provides session context)
       - chmod +x the script

    2. Register the hook in `~/.claude/settings.json`:
       - Add a new entry to the SessionStart hooks array AFTER worklog-digest.sh
       - Format: `{ "hooks": [{ "type": "command", "command": "~/.claude/hooks/risks-digest.sh" }] }`

    3. Register the MCP server with Claude Code:
       - Run: `claude mcp add --transport stdio --scope user --env MC_API_URL=http://100.x.x.x:3000 mission-control -- node /Users/ryanstern/mission-control/packages/mcp/dist/index.js`
       - Verify with: `claude mcp list`

    4. If portfolio-dashboard MCP server is currently registered, remove it:
       - Run: `claude mcp remove portfolio-dashboard --scope user` (or whatever scope it was added with)
       - This is the "direct swap" per user decision
  </action>
  <verify>
    <automated>MC_API_URL=http://100.x.x.x:3000 bash ~/.claude/hooks/risks-digest.sh && claude mcp list</automated>
  </verify>

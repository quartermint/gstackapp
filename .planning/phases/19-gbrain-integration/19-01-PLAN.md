<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gstackapp -> gbrain MCP server | SSH transport to Mac Mini; gbrain responses are untrusted external data |
| Operator request text -> gbrain query | User-provided text used as search query input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Tampering | gbrain response parsing | mitigate | Zod validation on all parsed responses in types.ts; malformed data returns empty results |
| T-19-02 | Tampering | SSH command injection | mitigate | SSH command is hardcoded in client.ts; request text passed only as MCP tool arguments (JSON), never shell args |
| T-19-03 | Denial of Service | SSH connection hang | mitigate | ConnectTimeout=5 on SSH; GbrainClient.connect() timeout prevents pipeline blocking |
| T-19-04 | Denial of Service | Slow gbrain queries | mitigate | Prefetch is fire-and-forget (not awaited); pipeline never blocks on gbrain |
| T-19-05 | Information Disclosure | gbrain content in logs | accept | gbrain data is internal knowledge (not sensitive PII); warn-level logs for parse failures are acceptable |
</threat_model>

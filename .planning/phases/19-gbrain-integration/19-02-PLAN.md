<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gbrain cached data -> Claude system prompt | Cached gbrain content injected into prompt; treated as context, not instructions |
| gbrain cached data -> request.json | Knowledge context written to file consumed by subprocess |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-06 | Tampering | gbrain entities in system prompt | mitigate | Entity excerpts are appended as context data, not executable instructions; Claude treats them as reference material. Zod validation in Plan 01 ensures well-formed data. |
| T-19-07 | Information Disclosure | knowledgeContext in request.json | accept | request.json is in /tmp with restricted permissions; knowledge data is internal project context, not sensitive PII |
| T-19-08 | Denial of Service | getGbrainCache() before clarification | accept | Single DB query per clarification call; negligible performance impact on Neon Postgres |
</threat_model>

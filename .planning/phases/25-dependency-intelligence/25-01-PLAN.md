<objective>
Build the dependency drift detection engine: pure functions that compare head commits across dependency pairs, severity escalation (info -> warning at 24h -> critical at 7d), and integration into the post-scan health phase pipeline. Also pipe `dependsOn` into the project API response and add an action hint for dependency_impact findings.

Purpose: This is the intelligence backbone of Phase 25 -- detecting when a dependency project advances and the dependent hasn't consumed those changes, then surfacing that as escalating health findings.
Output: Working drift detection in the scan cycle, dependsOn in API response, action hints for risk cards.

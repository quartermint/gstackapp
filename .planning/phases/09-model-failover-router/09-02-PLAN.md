    - BurnRateCalculator.shouldSwitch returns false when no DB, no billing cap, or no usage data
    - BurnRateCalculator.getCurrentBurnRate(provider) returns hourly/daily projection or null
    - BurnRateCalculator.recordPrediction(provider, predictedExhaustionTime) stores prediction for later accuracy check
    - BurnRateCalculator.checkPredictionAccuracy(provider) returns delta in minutes between predicted and actual cap hit, or null if no prediction recorded (per D-19)
    - RequestQueue.enqueue returns a Promise that resolves when the request is eventually processed
    - RequestQueue.enqueue throws AllProvidersDegradedError when queue full
    - RequestQueue.drain processes all queued requests through a provider
    - RequestQueue.clear rejects all with AllProvidersDegradedError
    - ModelRouter implements LLMProvider (has name property and createCompletion method)
    - ModelRouter.name returns 'router(primaryProviderName)'
    - On successful createCompletion, records token usage via UsageBuffer
    - On 429/billing error (detected by isProviderCapError): marks provider as degraded with cooldown, retries on next provider in chain
    - When fallbackPolicy is 'none': throws ProviderDegradedError immediately on first provider failure (no failover)
    - When fallbackPolicy is 'aggressive': tries every provider in chain regardless of quality tier
    - When fallbackPolicy is 'quality-aware':
      - Opus-capable providers defined as: `const OPUS_CAPABLE_PROVIDERS = ['anthropic']` -- only Anthropic has Opus-equivalent models
      - Opus-tier stages defined as: `const OPUS_TIER_STAGES = ['ceo', 'security']` -- maps to PROFILES where these stages use Opus models
      - For opus-tier stages, if next available provider is NOT in OPUS_CAPABLE_PROVIDERS: enqueue request rather than degrade quality
      - For non-opus-tier stages (eng, design, qa): accept any available provider
    - Predictive check before each request: if burn rate projects cap exhaustion within threshold, skip to next provider preemptively
    - All providers degraded + fallbackPolicy != 'none': queues request, resolves when first cooldown expires
    - All providers degraded + queue full (maxQueueSize): rejects with AllProvidersDegradedError
    - Every routing decision logged via logger with structured fields: { event: 'route_decision', provider, reason, burnRate, predictionAccuracy, fallbackPolicy, queueDepth } (per D-18)
    - On reactive catch when provider actually hits cap: compute predictionAccuracy = burnRateCalc.checkPredictionAccuracy(provider), include in log. This is the D-19 prediction tracking -- log the delta between predicted exhaustion time and actual cap hit for model improvement
    - extractRetryAfterMs used to set cooldown from 429 response when available
    - Provider degradation uses Math.max(retryAfterMs, cooldownMinutes * 60 * 1000)
    - ModelRouter.shutdown() calls usageBuffer.shutdown()

         - async createCompletion(params: CompletionParams & { stage?: string }): Promise<CompletionResult>
           - Step 1 (Predictive, per D-02): For each provider in chain order, check burnRateCalc.shouldSwitch(). Select first non-degraded, non-predicted-to-exhaust provider.
           - Step 2 (Request): Call selected provider.createCompletion(params)
           - Step 3 (Record usage): usageBuffer.record(providerName, result.usage, params.stage)
           - Step 4 (Log): Log route_decision with structured fields per D-18. Include burnRate from getCurrentBurnRate(), predictionAccuracy: null (no cap hit on success)
           - Return result
           - CATCH block (Reactive, per D-04):
             - If isProviderCapError(err):
               - Compute predictionAccuracy = burnRateCalc.checkPredictionAccuracy(providerName) (per D-19 -- log delta between predicted and actual cap hit)
               - Calculate cooldownMs = Math.max(extractRetryAfterMs(err) ?? 0, config.cooldownMinutes * 60 * 1000)
               - Mark provider as degraded: degradedProviders.set(providerName, { until: Date.now() + cooldownMs, reason })
               - Log degradation event with predictionAccuracy field (D-18, D-19)
               - If fallbackPolicy === 'none': throw new ProviderDegradedError(providerName, reason)
               - If fallbackPolicy === 'quality-aware':
                 - Determine stage tier: OPUS_TIER_STAGES.includes(params.stage ?? '') -> 'opus-tier', else 'any-tier'
                 - Find next non-degraded provider in chain
                 - If stage is opus-tier and !OPUS_CAPABLE_PROVIDERS.includes(nextProvider): enqueue instead of degrading
                 - Otherwise: retry on next provider
               - If fallbackPolicy === 'aggressive': try next non-degraded provider in chain
               - If all degraded: enqueue request (unless policy is 'none')
             - If not a cap error: re-throw
         - private isProviderDegraded(name: string): boolean
           - Check degradedProviders map, auto-clear if cooldown expired
           - When clearing expired degradation, drain queued requests through recovered provider
         - shutdown(): void -- usageBuffer.shutdown(), clear intervals

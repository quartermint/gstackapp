1. **Create stream bridge** (`packages/api/src/agent/stream-bridge.ts`):

Define the SSE event types the frontend will consume:
```typescript
export type AgentSSEEvent =
  | { type: 'text_delta'; text: string }
  | { type: 'tool_start'; name: string; id: string; input?: string }
  | { type: 'tool_result'; id: string; output: string; isError: boolean }
  | { type: 'turn_complete'; messageId: string }
  | { type: 'result'; sessionId: string; sdkSessionId: string; cost?: number; tokenUsage?: number }
  | { type: 'compact'; message: string }
  | { type: 'error'; message: string }
```

Export a `bridgeToSSE(rawMessage: any): AgentSSEEvent | null` function that maps raw SDK generator messages to these typed events. The SDK yields objects with a `type` field. Handle:
- `type === 'assistant'` with `content[].type === 'text'` -> emit `text_delta` for streaming text
- `type === 'assistant'` with `content[].type === 'tool_use'` -> emit `tool_start`
- `type === 'result'` -> emit `result` with session info and cost
- Tool results (within the generator flow) -> emit `tool_result`
- SDK streaming events when `includePartialMessages` is true -> handle `content_block_delta` etc.

Note: The exact SDK message shape should be discovered by reading the SDK types at implementation time. The bridge must handle both full messages and partial streaming events. Log unhandled message types at debug level for iterative discovery.

2. **Create agent loop wrapper** (`packages/api/src/agent/loop.ts`):

```typescript
import { query } from '@anthropic-ai/claude-agent-sdk'
import { gstackToolServer } from './tools'
import { buildSystemPrompt } from './system-prompt'
import { bridgeToSSE, type AgentSSEEvent } from './stream-bridge'

interface AgentLoopOptions {
  prompt: string
  sessionId?: string         // Our Drizzle session ID
  sdkSessionId?: string      // SDK session ID for resume
  projectPath?: string       // Project context
  maxTurns?: number          // Default 100
  maxBudgetUsd?: number      // Default 5.0
}

export async function* runAgentLoop(options: AgentLoopOptions): AsyncGenerator<AgentSSEEvent> {
  const systemPrompt = buildSystemPrompt({ projectPath: options.projectPath })

  for await (const message of query({
    prompt: options.prompt,
    options: {
      systemPrompt,
      resume: options.sdkSessionId,
      includePartialMessages: true,
      allowedTools: [
        'Read', 'Edit', 'Write', 'Bash', 'Glob', 'Grep',
        'mcp__gstack__*',
      ],
      permissionMode: 'bypassPermissions',   // Per D-11: personal workstation
      maxTurns: options.maxTurns ?? 100,
      maxBudgetUsd: options.maxBudgetUsd ?? 5.0,
      mcpServers: { gstack: gstackToolServer },
    }
  })) {
    const event = bridgeToSSE(message)
    if (event) yield event
  }
}
```

The function is an async generator (per D-01) that wraps the SDK's query() generator and yields typed SSE events.

3. **Create agent route** (`packages/api/src/routes/agent.ts`):

Hono app with:
- `GET /stream` — SSE endpoint. Query params: `prompt` (required for new messages), `sessionId` (our Drizzle session ID for resume). 
  - If sessionId provided, look up sdkSessionId and projectPath from Drizzle sessions table.
  - If no sessionId, create a new Drizzle session record (nanoid id, title from first 50 chars of prompt).
  - Call `runAgentLoop()` generator and pipe each yielded event through `streamSSE()`.
  - On `result` event: update the Drizzle session record with sdkSessionId, messageCount increment, tokenUsage, costUsd, lastMessageAt.
  - Also insert a message record for the user prompt (role: 'user') and a summary message for the assistant response (role: 'assistant').
  - Heartbeat: send empty heartbeat event every 15 seconds (same pattern as existing sse.ts).
  - Handle errors: wrap in try/catch, emit error SSE event, close stream.

- `POST /send` — Alternative to query param: accept JSON body `{ prompt: string, sessionId?: string }`. Redirect to GET /stream with params. This supports the frontend sending messages via fetch POST that opens an SSE stream.

**2. Create `packages/api/src/services/session-service.ts`:**

Export the following functions:

**`resolveProjectFromCwd(cwd: string, config: MCConfig): string | null`**
- Iterate `config.projects` array
- For each project: collect all paths (check `"copies" in project` for multi-copy entries with `project.copies[].path`, otherwise use `project.path`)
- For each path: check exact match (`cwd === path`) OR prefix match (`cwd.startsWith(path + "/")`)
- Track longest match (most specific wins): `bestMatch = { slug, pathLength }`
- Return `bestMatch?.slug ?? null`
- No git remote fallback in this implementation (per research recommendation -- skip for initial impl)

**`shouldDebounceHeartbeat(sessionId: string): boolean`**
- Module-level `Map<string, number>` called `heartbeatTimestamps`
- Constant `DEBOUNCE_MS = 10_000` (10 seconds)
- Check: if `heartbeatTimestamps.has(sessionId)` and `Date.now() - heartbeatTimestamps.get(sessionId)! < DEBOUNCE_MS`, return `true`
- Otherwise return `false` (caller should proceed with heartbeat)

**`recordHeartbeat(sessionId: string): void`**
- Set `heartbeatTimestamps.set(sessionId, Date.now())`

**`clearHeartbeatDebounce(sessionId: string): void`**
- `heartbeatTimestamps.delete(sessionId)` -- called on session stop to clean up

**`getBufferedFiles(sessionId: string): string[]`** and **`bufferFile(sessionId: string, filePath: string): void`**
- Module-level `Map<string, Set<string>>` called `fileBuffer`
- `bufferFile`: adds filePath to the Set for sessionId (creates Set if absent)
- `getBufferedFiles`: returns `[...fileBuffer.get(sessionId) ?? []]` and clears the entry
- Purpose: when heartbeat is debounced, file paths are buffered. On next non-debounced heartbeat, buffered files are flushed along with current file.

**`reapAbandonedSessions(db: DrizzleDb): number`**
- Import `eq`, `and`, `sql`, `lt` from `drizzle-orm`
- Import `sessions` from `../db/schema.js`
- Import `updateSessionStatus` from `../db/queries/sessions.js`
- Import `eventBus` from `./event-bus.js`
- Constant `REAPER_THRESHOLD_MS = 15 * 60 * 1000` (15 minutes)
- `const cutoff = new Date(Date.now() - REAPER_THRESHOLD_MS)`
- Query: `db.select().from(sessions).where(and(eq(sessions.status, "active"))).all()`
- Filter in JS: for each session, compute `lastActivity = session.lastHeartbeatAt ?? session.startedAt`. If `lastActivity.getTime() < cutoff.getTime()`, call `updateSessionStatus(db, session.id, "abandoned", "no heartbeat for 15+ minutes")`
- Emit `eventBus.emit("mc:event", { type: "session:abandoned", id: session.id })` for each reaped session
- Return count of reaped sessions
- Wrap entire function in try/catch, log errors, never throw

**`startSessionReaper(db: DrizzleDb, intervalMs: number = 180_000): ReturnType<typeof setInterval>`**
- Returns `setInterval(() => { try { const reaped = reapAbandonedSessions(db); if (reaped > 0) console.log(\`Session reaper: marked ${reaped} session(s) as abandoned\`); } catch (err) { console.error("Session reaper failed:", err); } }, intervalMs)`

**Endpoints (chained via `new Hono()`):**

**`POST /sessions/hook/start`** -- Claude Code SessionStart handler
- Validate with `hookStartSchema`
- Extract: `session_id`, `cwd`, `model`, `source`
- `const db = getInstance().db`
- Check for resume: try `getSession(db, hook.session_id)` in try/catch. If session exists AND status is "active", call `updateSessionHeartbeat(db, hook.session_id)` and return `c.json({ session: existing })` (200, not 201)
- If session exists but is completed/abandoned, treat as new session (session_id reuse after restart)
- Resolve project: `const config = getConfig(); const projectSlug = config ? resolveProjectFromCwd(hook.cwd, config) : null`
- Create: `const session = createSession(db, { sessionId: hook.session_id, source: "claude-code", model: hook.model ?? null, cwd: hook.cwd }, projectSlug)`
- Async event: `queueMicrotask(() => { eventBus.emit("mc:event", { type: "session:started", id: session.id }); })`
- Return `c.json({ session }, 201)`
- Wrap in try/catch: if AppError, return JSON error with status. Otherwise rethrow.

**`POST /sessions/hook/heartbeat`** -- Claude Code PostToolUse handler
- Validate with `hookHeartbeatSchema`
- Extract file path: `const filePath = hook.tool_input?.file_path`
- Always buffer the file path if present: `if (filePath) bufferFile(hook.session_id, filePath)`
- Check debounce: `if (shouldDebounceHeartbeat(hook.session_id)) return c.json({ debounced: true })`
- Record heartbeat timestamp: `recordHeartbeat(hook.session_id)`
- Get buffered files: `const filesTouched = getBufferedFiles(hook.session_id)`
- If filesTouched.length > 0 or file path exists, merge: `const files = filesTouched.length > 0 ? filesTouched : filePath ? [filePath] : undefined`
- Try: `updateSessionHeartbeat(db, hook.session_id, files)` -- catch silently (session may not exist yet if hook fires before start processes)
- Return `c.json({ ok: true })`

**`POST /sessions/hook/stop`** -- Claude Code Stop handler
- Validate with `hookStopSchema`
- Try: `const session = updateSessionStatus(db, hook.session_id, "completed")`
- `clearHeartbeatDebounce(hook.session_id)`
- Async: `queueMicrotask(() => { eventBus.emit("mc:event", { type: "session:ended", id: session.id }); })`
- Return `c.json({ session })`
- Catch: if session not found, return `c.json({ ok: true })` (stop for unknown session is not an error -- hook may fire for sessions started before MC)

**`GET /sessions`** -- Canonical list endpoint
- Validate query with `listSessionsQuerySchema`
- `const result = listSessions(db, query)`
- Return `c.json(result)`
- Standard AppError error handling

Close factory: `return new Hono().post(...)...get(...)`

  <behavior>
    buildDiffLineMap:
    - Parses unified diff patch format and returns Set of "filePath:lineNumber" strings
    - Correctly handles @@ hunk headers with +newStart format
    - Addition lines (+) and context lines (space) increment right-side line counter and are valid targets
    - Deletion lines (-) do NOT create valid targets on the right side
    - Files with no patch (binary files) produce no valid lines
    - Multiple hunks in same file are handled correctly

    mapFindingsToInlineComments:
    - Only includes findings with severity 'critical' or 'notable' (per D-07)
    - Only includes findings that have filePath AND lineStart
    - Only includes findings whose filePath:lineStart is in the validLines set (preventing 422 errors)
    - Returns at most 15 comments (per rate limit safety cap)
    - Critical findings are prioritized over notable when capping at 15
    - Each comment body includes stage identity label (e.g. "🔴 Security | **CRITICAL**") per D-08
    - Each comment body includes finding title, description, and suggestion if present
    - Minor findings are excluded entirely

    createInlineReview:
    - Does nothing if comments array is empty (no API call made)
    - Calls octokit.pulls.createReview with event 'COMMENT' and all comments batched
    - Uses line + side ('RIGHT') parameters, NOT deprecated position parameter
    - Returns array of review comment IDs for ghReviewCommentId storage
  </behavior>
  <action>
    1. Create `packages/api/src/github/inline-review.ts`:

       Define types:
       ```
       interface InlineComment { path: string; line: number; side: 'LEFT' | 'RIGHT'; body: string }
       interface DiffFile { filename: string; patch?: string }
       type FindingWithStage = Finding & { stage: string; id: string }
       ```

       Define STAGE_LABELS map matching comment-renderer.ts:
       `{ ceo: '🟠 CEO', eng: '🔵 Eng', design: '🟣 Design', qa: '🟢 QA', security: '🔴 Security' }`

       `MAX_INLINE_COMMENTS = 15` — cap per rate limit safety (per research open question 4)

       Export `function buildDiffLineMap(diffFiles: DiffFile[]): Set<string>`:
       - For each file with a patch, parse lines
       - Track rightLine counter starting from hunk header `@@ -oldStart,oldCount +newStart,newCount @@`
       - For `+` and ` ` (space) lines: add `${file.filename}:${rightLine}` to set, increment rightLine
       - For `-` lines: do NOT add to set, do NOT increment rightLine
       - Return the Set

       Export `function mapFindingsToInlineComments(findings: FindingWithStage[], diffFiles: DiffFile[]): InlineComment[]`:
       - Build validLines via buildDiffLineMap
       - Filter findings: severity is 'critical' or 'notable', has filePath and lineStart, filePath:lineStart is in validLines
       - Sort: critical first, then notable (so cap preserves highest severity)
       - Slice to MAX_INLINE_COMMENTS
       - Map each to InlineComment: path=filePath, line=lineStart, side='RIGHT', body=formatInlineComment(finding)

       Internal `function formatInlineComment(finding: FindingWithStage): string`:
       - Per D-08: starts with `${STAGE_LABELS[finding.stage]} | ${severity === 'critical' ? '**CRITICAL**' : 'Notable'}`
       - Then: `**${finding.title}**`
       - Then: finding.description (truncated to 500 chars)
       - If finding.suggestion: `**Suggestion:** ${finding.suggestion}`

       Export `async function createInlineReview(octokit: Octokit, owner: string, repo: string, prNumber: number, commitSha: string, comments: InlineComment[]): Promise<number[]>`:
       - If comments.length === 0, return empty array
       - Call `octokit.pulls.createReview({ owner, repo, pull_number: prNumber, commit_id: commitSha, event: 'COMMENT', comments: comments.map(c => ({ path: c.path, line: c.line, side: c.side, body: c.body })) })`
       - Extract review comment IDs from the response (via `octokit.pulls.listReviewComments({ owner, repo, pull_number: prNumber, review_id: review.data.id })` if needed, or from the review response)
       - Return the comment IDs for storing as ghReviewCommentId on findings

  <behavior>
    Feedback API:
    - POST /api/feedback accepts { findingId: string, vote: 'up'|'down', note?: string }
    - Returns 200 with { success: true } when finding exists and feedback is stored
    - Returns 404 with { error: 'Finding not found' } when findingId does not match any finding
    - Returns 400 with validation error when body fails FeedbackSubmissionSchema
    - Stores feedbackVote, feedbackNote, feedbackSource='dashboard', feedbackAt=now on the finding row
    - Overwrites previous feedback if called again for same finding

    Reaction Polling:
    - syncReactionFeedback queries findings with ghReviewCommentId and no feedbackVote
    - For each, calls octokit.reactions.listForPullRequestReviewComment
    - If thumbsUp > thumbsDown: sets feedbackVote='up', feedbackSource='github_reaction'
    - If thumbsDown > thumbsUp: sets feedbackVote='down', feedbackSource='github_reaction'
    - If no reactions or tied: skips (no update)

    Index.ts:
    - Feedback route mounted at app.route('/api', feedbackApp)
  </behavior>
  <action>
    1. Create `packages/api/src/routes/feedback.ts`:
       - Import `Hono` from `hono`
       - Import `db` from `../db/client`

       - Import `eq` from `drizzle-orm`
       - Import `FeedbackSubmissionSchema` from `@gstackapp/shared`

       Create Hono sub-app:
       ```
       const feedbackApp = new Hono()
       ```

       POST `/feedback` handler:
       - Parse and validate body with `FeedbackSubmissionSchema.safeParse(await c.req.json())`
       - If parse fails: return `c.json({ error: 'Invalid request', details: result.error.issues }, 400)`
       - Query finding by id: `db.query.findings.findFirst({ where: eq(findings.id, data.findingId) })`
       - If not found: return `c.json({ error: 'Finding not found' }, 404)`
       - Update finding: `db.update(findings).set({ feedbackVote: data.vote, feedbackNote: data.note ?? null, feedbackSource: 'dashboard', feedbackAt: new Date() }).where(eq(findings.id, data.findingId)).run()`
       - Return `c.json({ success: true })`

       Export default feedbackApp

    2. Add `syncReactionFeedback` function to `packages/api/src/github/comment.ts` (appending to existing file from Plan 01):
       - Import `isNotNull`, `isNull` from `drizzle-orm` (add to existing imports)
       - Export `async function syncReactionFeedback(octokit: Octokit, owner: string, repo: string): Promise<number>`:
         - Query findings where ghReviewCommentId is not null and feedbackVote is null
         - For each finding, call `octokit.reactions.listForPullRequestReviewComment({ owner, repo, comment_id: finding.ghReviewCommentId })`
         - Count thumbsUp (+1 reactions) and thumbsDown (-1 reactions)
         - If thumbsUp > thumbsDown: update feedbackVote='up', feedbackSource='github_reaction', feedbackAt=new Date()
         - If thumbsDown > thumbsUp: update feedbackVote='down', feedbackSource='github_reaction', feedbackAt=new Date()
         - If equal or no reactions: skip
         - Return count of findings updated
         - Wrap individual reaction calls in try/catch (a deleted comment returns 404, skip gracefully)
       - NOTE: This function is called on next pipeline run for the same repo, NOT on a schedule (per research recommendation for v1).

    - Add `useStaleCaptures(): { captures: CaptureItem[]; loading: boolean; refetch: () => void }`
    - Fetches from `GET /api/captures/stale` (created in Plan 01)
    - Returns full capture objects for the triage view (not just count)

    **3. Create triage view** (`packages/web/src/components/triage/triage-view.tsx`):
    - Props: `open: boolean; onClose: () => void; projects: ProjectItem[]`
    - Renders as a modal overlay (fixed position, dark scrim, centered card -- similar pattern to command palette)
    - Internal state: fetches stale captures via `useStaleCaptures()`, tracks current index
    - **One capture at a time flow:**
      - Show current capture: rawContent (full, not truncated), relative time ("captured 3 weeks ago"), current project assignment (if any)
      - If capture has link metadata: show link card preview
      - Three action buttons at the bottom:
        1. **Act** -- opens a small project selector dropdown (reuse pattern from CaptureCorrection). Selecting a project PATCHes the capture's projectId and advances to next.
        2. **Archive** -- PATCHes status to "archived" via `PATCH /api/captures/:id { status: "archived" }`. Capture leaves project cards but stays in FTS5 search. Advances to next.
        3. **Dismiss** -- DELETEs the capture via `DELETE /api/captures/:id`. Permanently removes it. Advances to next.
    - After each action, advance to the next stale capture. If no more captures remain, show a "All caught up!" message and close button.
    - Progress indicator: "3 of 12" or similar at the top
    - Esc closes the triage view (user can come back later)
    - Styling: centered modal, generous padding, readable text size (this is a focused reading view, not a compact list). Warm background, clear action buttons with distinct colors:
      - Act: terracotta (primary action)
      - Archive: warm-gray/slate-blue (neutral)
      - Dismiss: rust (destructive but not alarming)

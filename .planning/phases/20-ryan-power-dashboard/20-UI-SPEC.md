## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| 2xs | 2px | Hairline dividers, connector stroke width |
| xs | 4px | Icon gaps, inline padding, badge padding, stage node internal spacing |
| sm | 8px | Compact element spacing, card internal gaps, chart axis padding |
| md | 16px | Default element spacing, card padding, section gaps within views |
| lg | 24px | Section padding, panel gutters, sidebar-to-content gap |
| xl | 32px | Layout gaps, page padding (px-8 py-8 matching existing DashboardView) |

| 3xl | 64px | Page-level spacing (not expected in this phase) |

Exceptions: none

Source: DESIGN.md spacing scale (already in `app.css` as Tailwind tokens)

## Typography

This phase uses the full DESIGN.md scale since it is a data-dense power dashboard. All roles below appear:

| Role | Size | Weight | Line Height | Font | Usage in this phase |
|------|------|--------|-------------|------|---------------------|
| Display 2 | 32px | 600 (semibold) | 1.2 | General Sans | View titles ("Projects", "Pipeline Topology", "Knowledge Console") |
| Heading | 24px | 600 (semibold) | 1.2 | General Sans | Section headings within views, entity names in gbrain console |
| Subheading | 18px | 500 (medium) | 1.4 | Geist | Card titles (project names in overview), chart titles |
| Body | 15px | 400 (regular) | 1.6 | Geist | Descriptions, entity summaries, search results, insight explanations |
| Small | 13px | 400 (regular) | 1.5 | Geist | Status text, metadata, chart axis labels, secondary card info |
| Caption | 12px | 500 (medium) | 1.4 | Geist | Timestamps, relative dates ("3 hours ago"), health score labels |
| Mono Label | 11px | 500 (medium) | 1.4 | JetBrains Mono | Section headers, stage labels, repo full names, uppercase + 0.06em tracking |
| Code | 14px | 400 (regular) | 1.7 | JetBrains Mono | File paths, code snippets in cross-repo insights, gbrain query input |

Primary weights: 400 (regular) + 600 (semibold). Medium (500) used sparingly for Geist subheading/caption and JetBrains Mono labels per DESIGN.md scale.

Source: DESIGN.md typography scale

## Interaction Contracts

### Navigation: Extended Sidebar

The `AppView` type union extends from `'dashboard' | 'trends' | 'repos' | 'ideation' | 'autonomous' | 'operator'` to include:

| New View | Sidebar Label | Condition |
|----------|--------------|-----------|
| `'topology'` | "Topology" | Admin role only |
| `'knowledge'` | "Knowledge" | Admin role only |
| `'intelligence'` | "Intelligence" | Admin role only |

Existing "Dashboard" nav item now renders `ProjectOverview` (DASH-01) instead of the old `DashboardView` for admin users. The old dashboard content (PipelineHero + PRFeed) is accessed via the "Topology" view or project drill-in.

Sidebar rendering order for admin: Dashboard, Topology, Ideation, Intelligence, Knowledge, Trends, Repositories.

### Flow: Project Overview (DASH-01)

```
ProjectOverview
  |-- Health summary bar (3-segment: healthy/attention/critical)
  |-- ProjectGrid (enhanced, all quartermint repos)
  |     each ProjectCard shows:
  |       - StatusDot + project name
  |       - HealthBadge (score 0-100)
  |       - GSD phase progress
  |       - Last activity (relative timestamp)
  |       - Git branch + uncommitted count
  |     click --> ProjectDetailDrawer slides in from right
  |
  |-- ProjectDetailDrawer (480px right panel)
  |     - Project name + health score
  |     - Recent pipeline runs (last 5, each with VerdictBadge)
  |     - Git status detail
  |     - GSD phase detail
  |     - "Run Pipeline" accent button
  |     - Close: X button or click overlay or Escape key
```

### Flow: Pipeline Topology (DASH-02)

```
TopologyView
  |-- TopologyFilterBar
  |     - Repo multi-select dropdown
  |     - Status filter pills: All | Running | Complete | Flagged
  |     - "Run Pipeline" accent CTA (right-aligned)
  |
  |-- Scrollable canvas (horizontal + vertical)
  |     per active pipeline:
  |       repo name (mono label) | PipelineTopology (5 stage nodes + connectors)
  |       SSE updates stage states in real time
  |       click stage node --> shows FindingGroup for that stage
  |
  |-- Empty state: "No active pipelines" + "Run Pipeline" button
```

### Flow: Ideation Workspace (DASH-03)

```
IdeationWorkspace
  |-- Flow diagram header (horizontal)
  |     [Office Hours] --> [CEO Review] --> [Eng Review] --> [Execution]
  |     Each step: FlowStepNode (circle + label)
  |     Active step pulses with stage identity color
  |
  |-- IdeationView (existing component, unchanged)
  |     Input, pipeline, artifacts panels
```

### Flow: gbrain Console (DASH-04)

```
GbrainConsole (two-column: 55% / 45%)
  |-- Left: Search + Results
  |     GbrainSearchInput (text input + "Search" button)
  |     Results list: GbrainResultCard[] (scrollable)
  |     click result --> populates right column
  |
  |-- Right: Entity Detail
  |     GbrainEntityDetail
  |       - Entity name + type badge
  |       - Summary text
  |       - GbrainRelationshipGraph (radial SVG)
  |       - Related entities (clickable, navigate within console)
  |       - Compiled truth (scrollable block)
  |       - Sources
  |
  |-- Empty state (no search yet): "Query your knowledge base" + prompt text
  |-- Empty results: "No results for '{query}'" + suggestion text
  |-- gbrain unavailable: Warning alert card with degradation message
```

### Flow: Cross-Repo Intelligence (DASH-05)

```
IntelligenceView
  |-- Header: "Cross-Repo Intelligence" + count badge
  |
  |-- Section: "Active Alerts" (mono label)
  |     CrossRepoInsight cards, sorted by recency
  |     Each card expandable to show full match detail
  |
  |-- Section: "Pattern Detection" (mono label)
  |     PatternCard[] grouped by pattern type
  |     Each expandable to show individual matches
  |
  |-- Empty state: "No cross-repo patterns detected yet"
```

## Motion Contract

| Animation | Trigger | Duration | Easing | CSS |
|-----------|---------|----------|--------|-----|
| Card entrance | New card appears in any view | 250ms | ease-out | `animate-[fadeIn_250ms_ease-out_both]` (existing keyframe) |
| Running pulse | Active pipeline stage node | 2s infinite | ease-in-out | `animate-[pulse-glow_2s_ease-in-out_infinite]` (existing keyframe) |
| Trace flow | Pipeline connector line | 2.5s loop | linear | `animate-[trace-flow_2.5s_linear_infinite]` (existing keyframe) |
| Drawer slide-in | Project detail drawer opens | 250ms | ease-out | `transform: translateX(100%) -> translateX(0)` |
| Drawer slide-out | Project detail drawer closes | 200ms | ease-in | `transform: translateX(0) -> translateX(100%)` |
| Verdict materialize | Health badge appears on card | 400ms | ease | opacity 20% to 100% (DESIGN.md dim-to-bright) |
| Button hover | User hovers any action button | 150ms | default | `transition-colors duration-150` (existing pattern) |
| Flow step pulse | Active ideation flow step | 2s infinite | ease-in-out | `animate-[pulse-glow_2s_ease-in-out_infinite]` with stage color |
| Skeleton pulse | Loading state | continuous | default | `animate-pulse` (existing Tailwind) |

No bounce, no parallax, no scroll-driven animation. Per DESIGN.md anti-patterns.

    Convention scanner (convention-scanner.test.ts):
    - must_not_match rule: returns finding when pattern matches content
    - must_not_match rule: returns empty array when pattern does not match
    - negativeContext: suppresses finding when any negative context pattern matches content
    - must_match rule: returns finding when required pattern is ABSENT from content
    - must_match rule: returns empty array when required pattern IS present
    - overrides: skips rules listed in project overrides array
    - aggregation: returns single HealthFindingInput with aggregated detail when multiple rules violated
    - severity: uses worst severity across violated rules (critical > warning > info)
    - invalid regex: skips rule without crashing (try/catch around new RegExp)
    - empty rules: returns empty array
    - empty content: handles gracefully (must_not_match rules don't match, must_match rules fire)

    Config schema (extend config.test.ts):
    - conventions array validates with all fields (id, pattern, description, negativeContext, severity, matchType)
    - conventions defaults to empty array when omitted (backward compat)
    - conventionOverrides on project entry defaults to empty array when omitted
    - convention id validates kebab-case format
    - invalid convention rule (missing id/pattern) rejected by schema
  </behavior>

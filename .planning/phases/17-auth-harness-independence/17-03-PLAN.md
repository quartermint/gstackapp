<objective>
Build the Claude Code subprocess execution engine: spawn `claude -p` per request, monitor output via file polling, signal completion via localhost callback, stream progress to the web UI via SSE, and handle decision gates that pause/resume the pipeline.

Purpose: This is the harness independence layer — the web UI can trigger and monitor pipeline runs without the CLI harness package. Decision gates create a human-in-the-loop flow.
Output: End-to-end pipeline: submit request -> spawn Claude -> progress streams to UI -> gates pause for approval -> completion.
</objective>

    **System prompt builder** (packages/api/src/pipeline/system-prompt.ts):

    Export `buildPipelineSystemPrompt(pipelineId: string, outputDir: string, callbackUrl: string): string`. The system prompt instructs Claude Code to:
    1. Read `${outputDir}/request.json` for the user's request.
    2. Execute pipeline stages in order: clarify -> plan -> execute -> verify.
    3. Write progress files as `${outputDir}/progress-NNN.json` with schema `{ stage: string, status: 'running' | 'complete', message?: string, result?: object, timestamp: string }`. Increment NNN (zero-padded 3 digits) for each file.
    4. For decision gates: write `${outputDir}/gate-{nanoid}.json` with `{ id: string, title: string, description: string, options: string[] }`. Then poll for `${outputDir}/gate-{id}-response.json` containing `{ response: string }` before continuing.
    5. On completion: write `${outputDir}/result.json` with final output. Then call `${callbackUrl}` via `curl -X POST -H 'Content-Type: application/json' -d '{"pipelineId":"${pipelineId}"}' ${callbackUrl}`.

    **Spawner** (packages/api/src/pipeline/spawner.ts):

    Export `spawnPipeline(options: PipelineSpawnOptions): { pid: number }`. Per D-07: spawn `claude` CLI as child process with `child_process.spawn`. Args: `['-p', options.prompt, '--system-prompt', systemPrompt, '--allowedTools', 'Read,Write,Bash,Glob,Grep', '--max-turns', '50', '--output-format', 'json']`. Options: `{ cwd: options.projectPath, stdio: ['pipe', 'pipe', 'pipe'], env: { ...process.env }, detached: true }`. Call `child.unref()`. Create outputDir with `mkdirSync(outputDir, { recursive: true })`. Write `request.json` with `{ pipelineId, prompt: options.prompt, whatGood: options.whatGood, outputDir, callbackUrl }`. Log stderr via pino. Return `{ pid: child.pid }`.

    Interface:

    **File watcher** (packages/api/src/pipeline/file-watcher.ts):

    Export `watchPipelineOutput(pipelineId: string, outputDir: string): void`. Poll `outputDir` every 2 seconds (Claude's discretion on interval) using `fs.readdirSync`. Track which files have been processed (Set of filenames). For each new `progress-*.json` file: read content, emit via `pipelineBus.emit('pipeline:event', { type: 'stage:running' or 'stage:completed', runId: pipelineId, stage, verdict, timestamp })`. For each new `gate-*.json` (not response files): emit `pipelineBus.emit('pipeline:event', { type: 'operator:gate', runId: pipelineId, gateId, title, description, options, timestamp })`.

    Export `stopWatching(pipelineId: string): void` — clears the interval. Export `finalSweep(pipelineId: string, outputDir: string): void` — processes any unread files before emitting completion.

The route validates the pipelineId, triggers `finalSweep`, emits completion event, updates `operatorRequests.status = 'complete'` and `completedAt = new Date()`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hono API -> child_process.spawn | Server spawns external process with user-derived prompt |
| Claude Code subprocess -> filesystem | Subprocess writes to /tmp/pipeline-{id}/ |
| File system -> pipelineBus -> SSE -> Browser | Progress data flows from files through server to client |
| Browser -> POST gate-response -> filesystem | User response written to file for subprocess to read |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-14 | Tampering | spawner.ts | mitigate | Never interpolate user input into CLI args. User prompt passed via file (request.json per D-08), not CLI args. System prompt is server-generated. |
| T-17-15 | Tampering | file-watcher.ts | mitigate | Validate outputDir starts with `/tmp/pipeline-` prefix before reading. Only read .json files. Parse with try/catch. |
| T-17-16 | Tampering | gate-response route | mitigate | Validate gateId format (nanoid pattern). Validate outputDir exists and belongs to the request. Write response as JSON only. |
| T-17-17 | Denial of Service | spawner.ts | mitigate | --max-turns 50 caps subprocess execution. Track PID in DB. One active pipeline per user (check before spawn). Kill orphaned processes on server startup. |
| T-17-18 | Information Disclosure | file-watcher.ts | mitigate | Only watch directories matching /tmp/pipeline-{requestId}/ where requestId belongs to the authenticated user. No directory traversal. |
| T-17-19 | Denial of Service | callback route | mitigate | Callback route validates pipelineId exists in DB before processing. Internal-only (localhost), not exposed externally. |
| T-17-20 | Elevation of Privilege | spawner.ts | mitigate | Claude Code subprocess inherits server env but runs in /tmp workspace. --allowedTools restricts to Read,Write,Bash,Glob,Grep. No network tools. |
</threat_model>

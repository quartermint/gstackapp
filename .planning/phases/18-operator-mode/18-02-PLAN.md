Create `packages/api/src/pipeline/verification-reader.ts`:

```typescript
export interface VerificationReport {
  passed: boolean
  summary: string
  whatBuilt: string[]
  qualityChecks: { passed: number; total: number }
  filesChanged: number
  failureDetails?: string
}

export function readVerificationResult(outputDir: string): VerificationReport | null
```

Implementation:
- Read `${outputDir}/result.json` with try/catch (return null if missing).
- Parse JSON. Expected structure from Claude Code harness: `{ status: 'pass'|'fail'|'block', stages: [...], summary: string }`.
- Transform to VerificationReport:
  - `passed`: status === 'pass'
  - `summary`: Plain-language 1-2 sentence summary. If the result has a summary field, use it. Otherwise generate: "All quality checks passed. Your request has been completed." or "Some quality checks found issues."
  - `whatBuilt`: Extract from stages array — list items that were completed.
  - `qualityChecks`: Count stages with pass vs total.
  - `filesChanged`: Extract from result or default to 0.
  - `failureDetails`: If passed === false, extract the failure description in plain language.

Modify `packages/api/src/pipeline/file-watcher.ts`:

- After existing progress/gate file detection, add detection for `result.json`:
  - When detected, call `readVerificationResult(outputDir)`.
  - If result exists and `passed === true`: emit `pipelineBus.emit('pipeline:event', { type: 'operator:verification:report', runId: pipelineId, report: result, timestamp })`.
  - If result exists and `passed === false`: emit `pipelineBus.emit('pipeline:event', { type: 'operator:error', runId: pipelineId, errorType: 'verification-failure', message: 'Quality check found issues. Some checks didn\'t pass. You can request changes to address them or ask Ryan for help.', report: result, timestamp })`.
  - Log audit trail entry: `'verification_pass'` or `'verification_fail'`.

- Add detection for `error-*.json` files in the output directory:
  - Parse error JSON for type and message.
  - Emit `operator:error` SSE event with the appropriate errorType.

Tests: Create test fixtures for result.json (pass and fail variants). Test readVerificationResult parsing. Test file watcher detection of result.json. Mock filesystem operations.

## Typography

This phase uses a subset of the full DESIGN.md scale. Only these roles appear in Operator Mode:

| Role | Size | Weight | Line Height | Font | Usage in this phase |
|------|------|--------|-------------|------|---------------------|
| Heading | 24px | 600 (semibold) | 1.2 | General Sans | Page title "What can I help with?" |
| Body | 15px | 400 (regular) | 1.6 | Geist | Chat messages, form descriptions, error explanations |
| Small | 13px | 400 (regular) | 1.5 | Geist | Card body text, clarification questions, status messages |
| Mono Label | 11px | 400 (regular) | 1.4 | JetBrains Mono | Section headers ("In Progress", "Recent Requests"), uppercase, 0.06em tracking |

Weights used: 400 (regular) + 600 (semibold). JetBrains Mono's optical weight at 400 provides sufficient differentiation from Geist at the same weight — no medium (500) weight needed.

Source: DESIGN.md typography scale, existing component patterns in `OperatorHome.tsx`, `MessageBubble.tsx`

## Interaction Contracts

### Flow: Intake to Completion (single-page, no navigation)

```
OperatorHome
  |-- IntakeForm (existing)
  |     submit --> POST /api/operator/request
  |
  |-- ClarificationThread (new)
  |     system asks question (assistant bubble)
  |     operator types answer (InputArea)
  |     repeat up to 5x
  |     system generates ExecutionBrief
  |
  |-- ExecutionBrief (new)
  |     "Approve & Start" --> pipeline begins
  |     "Request Changes" --> return to ClarificationThread
  |
  |-- OperatorProgressBar (new)
  |     SSE events update steps: thinking -> planning -> building -> checking -> done
  |     Decision gates appear as inline ErrorCard-style cards
  |
  |-- VerificationReport (new, on completion)
  |     OR ErrorCard (on failure/timeout/ambiguity/provider exhaustion)
  |
  |-- AuditTrail (new, always available)
```

### Decision Gate Interaction (D-06)

| Action | Button Style | Behavior |
|--------|-------------|----------|
| Approve | accent border, accent text, hover bg-accent/10 | POST gate response, resume pipeline |
| Request Changes | border-border, text-muted, hover text-primary | POST gate response, pipeline receives change request |
| Ask Ryan | border-border, text-muted, hover text-primary | POST escalation, show "Escalated to Ryan" confirmation |

First option always styled as recommended (accent color). Matches existing `GateCard` pattern in `PipelineProgress.tsx`.

### Error State Interactions (D-04)

| Error Type | Left Border | Title | Buttons |
|------------|------------|-------|---------|
| Timeout (OP-08) | `#FFB020` (flag) | "Taking longer than expected" | [Keep Waiting] [Ask Ryan] |
| Verification Failure (OP-09) | `#FF5A67` (block) | "Quality check found issues" | [Request Changes (pre-selected)] [Ask Ryan] |
| Ambiguous Scope (OP-10) | `#FFB020` (flag) | "We couldn't fully pin down the scope" | [Ask Ryan] |
| Provider Exhaustion (OP-11) | `#6F7C90` (skip) | "Temporarily unavailable" | [Retry Later] |

### Progress Step States

| Step | Pending | Running | Complete |
|------|---------|---------|----------|
| Icon | 20px circle, border-border | 20px circle, border-running, inner dot, animate-pulse | 20px circle, bg-pass/20, checkmark SVG |
| Label color | text-muted/50 | text-primary | text-muted |
| Animation | none | 2s ease-in-out pulse-glow (from DESIGN.md) | none |

Source: Existing `StageIcon` in `PipelineProgress.tsx` (reuse pattern, adapt for horizontal layout)

## Motion Contract

| Animation | Trigger | Duration | Easing | CSS |
|-----------|---------|----------|--------|-----|
| Card entrance | New card appears in chat thread | 250ms | ease-out | `animate-[fadeIn_250ms_ease-out_both]` (existing keyframe) |
| Running pulse | Active progress step | 2s infinite | ease-in-out | `animate-[pulse-glow_2s_ease-in-out_infinite]` (existing keyframe) |
| Accordion expand | User clicks expandable section | 200ms | ease-out | `transition: max-height 200ms ease-out` |
| Button hover | User hovers action button | 150ms | default | `transition-colors duration-150` (existing pattern) |
| Verdict materialize | Verification report appears | 400ms | ease | opacity 20% to 100% (DESIGN.md dim-to-bright) |

No bounce, no parallax, no scroll-driven animation. Per DESIGN.md anti-patterns.

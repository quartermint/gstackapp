Build the sprint heatmap visualization and SSE real-time connection for the dashboard. The heatmap is the first thing users see after the capture field -- a GitHub-style contribution grid showing commit intensity per project over 12 weeks, making serial sprint patterns visible at a glance. Purpose: The heatmap provides the "serial sprint" pattern visibility -- you can literally see which project got intense focus and when. SSE makes the dashboard feel alive -- captures appear, enrichments update, scans refresh, all without touching F5.

       - Use `useRef` for the options to avoid stale closure issues (same pattern as useKeyboardShortcuts -- update ref in useEffect, read from ref in handlers). This is critical per research pitfall #4.

       - Each listener parses `e.data` as JSON and calls the corresponding option callback via the ref
       - Implement custom reconnection with exponential backoff on error:
         - Close EventSource on error
         - Calculate delay: `Math.min(1000 * Math.pow(2, retryCount), 30_000) + Math.random() * 1000` (jitter)
         - Increment retryCount, setTimeout to reconnect
         - Reset retryCount to 0 on successful connection (onopen)
       - Return cleanup function that closes EventSource
       - Do NOT show SSE connection state visually (per discretion -- health dot already shows API reachability)

    2. Create `packages/web/src/hooks/use-heatmap.ts`:
       - Fetch from "/api/heatmap" on mount

       - Return `{ data: HeatmapEntry[], loading: boolean, refetch: () => void }` where HeatmapEntry = `{ projectSlug: string, date: string, count: number }`
       - Include refetch via fetchCounter pattern (SSE scan:complete will trigger this)

    3. Extend `packages/web/src/hooks/use-projects.ts`:
       - Add fetchCounter state and refetch callback (same pattern as useCaptureCounts)
       - Add fetchCounter to useEffect dependency array
       - Return `refetch` in the return object: `{ groups, loading, error, refetch }`
       - This enables SSE scan:complete to trigger project list refresh

<objective>
Surface cross-repo intelligence in both PR comments and the dashboard detail view.

Purpose: XREP-03 requires cross-repo insights to appear in two places: (1) the GitHub PR comment as a "Seen in your other repos" callout section, and (2) the dashboard pipeline detail view with styled insight cards. This plan adds rendering to both surfaces, wires the search module from Plan 01 into the comment manager and pipelines API, and updates the bottom strip from placeholder to live data.

Output: PR comment cross-repo section, dashboard CrossRepoInsight component, updated pipelines API, live bottom strip.
</objective>

    Add new function `renderCrossRepoSection(matches: CrossRepoMatch[]): string`:
    - If matches is empty, return empty string
    - Render a section with header "### Seen in your other repos" (per D-05)
    - For each match, render a blockquote with:
      - **{title}** -- `{repo_full_name}` ({similarity}% similar)
      - File: `{file_path}` (if present)

    **Step 2: Wire cross-repo search into comment.ts**

    In `packages/api/src/github/comment.ts`, in `updatePRComment`:
    - Import `findCrossRepoMatches` and `embedTexts`, `normalizeFindingText` from `../embeddings`
    - Import `rawDb` from `../db/client`
    - After building `findingsWithStage` array, for each finding:
      - Normalize the finding text
      - Embed it via `embedTexts([normalizedText])`
      - Query `findCrossRepoMatches(rawDb, embedding, repoFullName)` where repoFullName comes from the pipeline run's repo
    - Collect all unique cross-repo matches across all findings (deduplicate by finding_id)
    - Pass `crossRepoMatches` to `renderComment` input

    Important: The cross-repo search needs the repo full name. The comment manager already knows `owner/repo` from the CommentInput. Construct `repoFullName = \`${owner}/${repo}\``.

    Optimization: Rather than embedding each finding individually at comment render time, query vec_findings directly using the finding IDs from this pipeline run. For each finding that has an embedding in vec_findings, run the KNN search. If no embeddings exist yet (embedding is still in-flight), skip gracefully. Wrap the entire cross-repo search in try/catch -- failure should NOT prevent the comment from rendering.

    **Step 3: Add crossRepoMatches to GET /api/pipelines/:id**

    In `packages/api/src/routes/pipelines.ts`:
    - Import `findCrossRepoMatches` from `../embeddings`
    - Import `rawDb` from `../db/client`
    - In the `GET /:id` handler, after fetching findings:
      - For each finding, check if it has an embedding in vec_findings (query by finding_id)
      - If yes, run findCrossRepoMatches with that embedding
      - Collect cross-repo matches and include in each finding's response
    - Add `crossRepoMatches` field to the response at the top level (array of matches across all findings)
    - Wrap in try/catch -- if vec_findings doesn't have data yet, return empty array

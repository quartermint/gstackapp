    - acquireLock writes JSON to ~/.gstackapp/sync.lock with pid, hostname, startedAt, paths
    - acquireLock throws if lock exists and PID is alive (message includes PID and hostname)
    - acquireLock removes stale lock (dead PID) and proceeds with console.warn
    - releaseLock deletes the lock file, no-ops if already gone
    - withLock(paths, fn) acquires lock, runs fn, releases lock even if fn throws
    - withLock installs SIGINT/SIGTERM handlers that release lock

    **packages/api/src/github/auth.ts** (per D-10, RESEARCH.md Pattern 3):
    - Import { Octokit } from '@octokit/rest'
    - Import { createAppAuth } from '@octokit/auth-app'
    - Import { config } from '../lib/config'
    - Create module-level Map<number, Octokit> for installationClients cache
    - Export function getInstallationOctokit(installationId: number): Octokit
      - Check cache first, return if exists
      - Create new Octokit with authStrategy: createAppAuth, auth: { appId: config.githubAppId, privateKey: config.githubPrivateKey, installationId }
      - Store in cache and return
    - Export function clearInstallationClient(installationId: number): void
      - Delete from cache (used when installation is deleted)
    - Note in comment: @octokit/auth-app internally caches up to 15K installation tokens and auto-refreshes at 59 min. No manual token management needed.

    **packages/api/src/lib/idempotency.ts** (per D-07, RESEARCH.md Pattern 2):
    - Import { db } from '../db/client'
    - Import { pipelineRuns, pullRequests } from '../db/schema'
    - Import { nanoid } from 'nanoid'
    - Import { sql, eq } from 'drizzle-orm'
    - Export async function ensurePullRequest(params: { repoId: number, number: number, title: string, authorLogin: string, headSha: string, baseBranch: string }): Promise<number>
      - Insert pullRequest with onConflictDoUpdate on (repoId, number) to update headSha, title, updatedAt
      - Return the PR's autoIncrement id (use RETURNING or query after insert)
    - Export function tryCreatePipelineRun(params: { deliveryId: string, prId: number, installationId: number, headSha: string }): { created: boolean, runId: string }
      - const runId = nanoid()
      - Use db.insert(pipelineRuns).values({ id: runId, deliveryId, prId, installationId, headSha, status: 'PENDING', createdAt: new Date() }).onConflictDoNothing() on delivery_id unique constraint
      - Check result.changes > 0 to determine if row was created
      - Return { created: result.changes > 0, runId: result.changes > 0 ? runId : '' }
    - CRITICAL: Use INSERT ON CONFLICT DO NOTHING, NOT query-then-insert (race condition per RESEARCH.md Anti-Patterns)

    Wire up the webhook endpoint, event handlers, and server entry point.

    **packages/api/src/github/webhook.ts** (per D-08, D-09, D-10, RESEARCH.md Pattern 1, Pitfall 1):
    - Import { Hono } from 'hono'
    - Import { Webhooks } from '@octokit/webhooks'
    - Import { config } from '../lib/config'
    - Import { registerHandlers } from './handlers'
    - Create webhooks instance: new Webhooks({ secret: config.githubWebhookSecret })
    - Call registerHandlers(webhooks)
    - Create webhookApp = new Hono()
    - POST /api/webhook route:
      1. Read raw body FIRST with `await c.req.text()` -- CRITICAL per Pitfall 1, before any JSON parsing
      2. Extract headers: x-github-delivery, x-hub-signature-256, x-github-event
      3. Validate all three headers present, return 400 if missing
      4. Call webhooks.verifyAndReceive({ id: deliveryId, name: eventName as any, signature, payload: rawBody })
         - Pass rawBody as string, NOT parsed JSON (per RESEARCH.md)
      5. Catch errors: return 401 for signature verification failures
      6. Return c.json({ ok: true }) immediately -- event handlers run async in the webhooks instance
    - Export default webhookApp
    - CRITICAL: Do NOT apply any JSON parsing middleware to this route. No bodyLimit, no json() middleware.

    **packages/api/src/github/handlers.ts** (per D-09, RESEARCH.md Code Example: Webhook Event Handlers, Pitfall 4):
    - Import { Webhooks } from '@octokit/webhooks'
    - Import { db } from '../db/client'
    - Import { githubInstallations, repositories } from '../db/schema'
    - Import { getInstallationOctokit, clearInstallationClient } from './auth'
    - Import { ensurePullRequest, tryCreatePipelineRun } from '../lib/idempotency'
    - Import { eq } from 'drizzle-orm'
    - Export function registerHandlers(webhooks: Webhooks):
      a) webhooks.on('installation.created', async ({ payload }) => { ... })
         - Upsert installation via db.insert(githubInstallations).values({ id: payload.installation.id, accountLogin: payload.installation.account.login, accountType: payload.installation.account.type, appId: payload.installation.app_id, status: 'active' }).onConflictDoUpdate({ target: githubInstallations.id, set: { status: 'active', updatedAt: new Date() } }).run()
         - Persist payload.repositories if present (iterate, db.insert(repositories).values({ id: repo.id, installationId: payload.installation.id, fullName: repo.full_name, isActive: true }).onConflictDoNothing().run())
         - Per Pitfall 4: If payload.installation.repository_selection === 'all', call getInstallationOctokit(payload.installation.id) then octokit.apps.listReposAccessibleToInstallation() and persist each repo
      b) webhooks.on('installation.deleted', async ({ payload }) => { ... })
         - db.update(githubInstallations).set({ status: 'deleted', updatedAt: new Date() }).where(eq(githubInstallations.id, payload.installation.id)).run()
         - Call clearInstallationClient(payload.installation.id)
      c) webhooks.on('installation_repositories.added', async ({ payload }) => { ... })
         - Iterate payload.repositories_added, insert each with onConflictDoNothing
      d) webhooks.on('installation_repositories.removed', async ({ payload }) => { ... })
         - Iterate payload.repositories_removed, update isActive: false
      e) webhooks.on(['pull_request.opened', 'pull_request.synchronize', 'pull_request.reopened'], async ({ id, payload }) => { ... })
         - const prId = await ensurePullRequest({ repoId: payload.repository.id, number: payload.pull_request.number, title: payload.pull_request.title, authorLogin: payload.pull_request.user.login, headSha: payload.pull_request.head.sha, baseBranch: payload.pull_request.base.ref })
         - const { created, runId } = tryCreatePipelineRun({ deliveryId: id, prId, installationId: payload.installation.id, headSha: payload.pull_request.head.sha })
         - If created: log "Pipeline run created: {runId} for PR #{payload.pull_request.number}"
         - If not created: log "Duplicate delivery ignored: {id}"
         - Note: Phase 2 will add actual pipeline dispatch here. For now, just persist.

      - githubPrivateKey: z.string()
      - githubWebhookSecret: z.string()

    - resolvePrivateKey() function: check GITHUB_PRIVATE_KEY_PATH first (readFileSync), fall back to GITHUB_PRIVATE_KEY with `\n` replacement, throw if neither
    - Export config = configSchema.parse({...process.env mappings...})
    - Export type Config = z.infer<typeof configSchema>

    **packages/api/src/db/schema.ts** (per D-03, RESEARCH.md Pattern 5):
    Exact 6-table schema from RESEARCH.md with these specifics:
    - githubInstallations: id (integer PK, GitHub installation ID), accountLogin (text NOT NULL), accountType (text NOT NULL), appId (integer NOT NULL), status (text NOT NULL default 'active'), createdAt (integer timestamp_ms with $defaultFn), updatedAt (integer timestamp_ms with $defaultFn)
    - repositories: id (integer PK, GitHub repo ID), installationId (integer NOT NULL FK to githubInstallations), fullName (text NOT NULL), defaultBranch (text NOT NULL default 'main'), isActive (integer boolean NOT NULL default true), createdAt (integer timestamp_ms). Index: repo_installation_idx on installationId
    - pullRequests: id (integer PK autoIncrement), repoId (integer NOT NULL FK to repositories), number (integer NOT NULL), title (text NOT NULL), authorLogin (text NOT NULL), headSha (text NOT NULL), baseBranch (text NOT NULL), state (text NOT NULL default 'open'), createdAt, updatedAt. UniqueIndex: pr_repo_number_idx on (repoId, number)
    - pipelineRuns: id (text PK, nanoid), deliveryId (text NOT NULL), prId (integer NOT NULL FK to pullRequests), installationId (integer NOT NULL), headSha (text NOT NULL), status (text NOT NULL default 'PENDING'), commentId (integer nullable), startedAt (integer timestamp_ms nullable), completedAt (integer timestamp_ms nullable), createdAt (integer timestamp_ms). UniqueIndex: delivery_id_idx on deliveryId. Indexes: pipeline_pr_idx on prId, pipeline_status_idx on status
    - stageResults: id (text PK, nanoid), pipelineRunId (text NOT NULL FK to pipelineRuns), stage (text NOT NULL), verdict (text NOT NULL default 'PENDING'), summary (text nullable), tokenUsage (integer nullable), durationMs (integer nullable), error (text nullable), createdAt, completedAt. Index: stage_pipeline_idx on pipelineRunId. UniqueIndex: stage_run_stage_idx on (pipelineRunId, stage)
    - findings: id (text PK, nanoid), stageResultId (text NOT NULL FK to stageResults), pipelineRunId (text NOT NULL FK to pipelineRuns), severity (text NOT NULL), category (text NOT NULL), title (text NOT NULL), description (text NOT NULL), filePath (text nullable), lineStart (integer nullable), lineEnd (integer nullable), suggestion (text nullable), codeSnippet (text nullable), createdAt. Indexes: finding_stage_idx, finding_pipeline_idx, finding_severity_idx

    Import from 'drizzle-orm/sqlite-core': sqliteTable, text, integer, index, uniqueIndex
    Import sql from 'drizzle-orm' (for raw SQL if needed)

    **packages/api/src/db/client.ts** (per D-04, D-05, RESEARCH.md Pattern 4):
    - Import Database from 'better-sqlite3'
    - Import { drizzle } from 'drizzle-orm/better-sqlite3'
    - Import * as schema from './schema'
    - Import { config } from '../lib/config'
    - Import { mkdirSync } from 'node:fs'

    - const sqlite = new Database(config.databasePath)

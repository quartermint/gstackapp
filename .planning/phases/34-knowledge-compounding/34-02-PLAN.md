**Per D-04 (Claude's discretion on significance heuristic):**

Create `packages/api/src/services/solution-extractor.ts` with:

1. **SessionSignal interface:**
```typescript
export interface SessionSignal {
  durationMinutes: number;
  filesCount: number;
  commitCount: number;
  projectSlug: string | null;
}
```

2. **isSignificantSession(signal: SessionSignal): boolean** -- Pure function implementing the heuristic from research:
   - No project slug -> false
   - Duration < 5 min -> false
   - filesCount < 3 AND commitCount === 0 -> false
   - commitCount >= 1 -> true
   - duration >= 30 AND filesCount >= 5 -> true
   - filesCount >= 10 -> true
   - Otherwise -> false

3. **buildSessionSignal(db: DrizzleDb, session)**: SessionSignal -- Builds signal from session data:
   - durationMinutes = (endedAt - startedAt) / 60000
   - filesCount = JSON.parse(session.filesJson ?? "[]").length
   - commitCount = count commits where projectSlug matches AND authorDate between session startedAt and endedAt. Use raw sql count query against commits table. Note: authorDate is stored as ISO string, startedAt/endedAt are Date objects (from Drizzle timestamp mode). Convert to ISO strings for comparison.
   - projectSlug = session.projectSlug

4. **buildSolutionContent(commits, files, session): string** -- Builds readable content from commit messages and file list:
   - Format: "## Session Summary\n\n**Project:** {slug}\n**Duration:** {min} minutes\n**Files:** {count}\n\n## Commits\n\n{commit messages}\n\n## Files Modified\n\n{file list}"
   - This becomes the solution `content` field for search indexing.

5. **buildTitle(commits): string** -- Extract title from commit messages:
   - If 1 commit: use its message (trim to 100 chars)
   - If multiple: use the first conventional commit prefix or the first commit message
   - If no commits: "Session work on {projectSlug}"

6. **extractSolutionMetadata(content, projectSlug)** -- LM Studio enrichment (async):
   - Check getLmStudioStatus().health === "ready", return null if not
   - Use generateText + Output.object with a Zod schema (solutionExtractionSchema) to extract: title, problemType, symptoms, rootCause, tags, severity, module
   - Follow the exact pattern from ai-query-rewriter.ts for calling LM Studio
   - System prompt: "You are analyzing a Claude Code session summary to extract structured solution metadata. Identify what problem was solved, what caused it, and how it was fixed."
   - Return the extracted metadata or null on any error

**1b. Create `runPostScanHealthPhase` function:**

Signature:
```typescript
async function runPostScanHealthPhase(
  healthDataMap: Map<string, HealthScanData>,
  db: DrizzleDb,
  sqlite: Database.Database
): Promise<void>
```

The function has three stages:

**Stage 1: Per-repo health checks + finding persistence**

For each entry in `healthDataMap`:
1. Call `runHealthChecks(healthScanData)` to get findings array
2. For each finding, call `upsertHealthFinding(db, sqlite, finding)` (preserves detectedAt on re-detection)
3. Collect the `checkType` values from the findings array as `activeCheckTypes`
4. Call `resolveFindings(sqlite, slug, activeCheckTypes)` to resolve any findings that no longer apply
   - Extract the project slug from the map key (format is `${slug}:${host}` -- split on `:` and take first part)
   - IMPORTANT: For multi-copy projects, resolveFindings should only resolve per-repo check types (not `diverged_copies` -- that is handled in Stage 2)
   - Add `"diverged_copies"` to activeCheckTypes when calling resolveFindings, so it does not get auto-resolved here

**Stage 2: Multi-copy divergence detection**

Group all scanned copies by their normalized remote URL:
1. Build a `Map<string, HealthScanData[]>` from healthDataMap, where key is `normalizeRemoteUrl(scanData.remoteUrl)` (skip entries with null remoteUrl)
2. Also load existing copy records from DB via `getCopiesByRemoteUrl` for each unique remote URL (this picks up copies from hosts that were not scanned in this cycle, e.g., Mac Mini was unreachable)
3. For each group with 2+ copies (combining fresh scan data and DB records):
   a. Compare HEAD commits between copies
   b. If all HEADs match: not diverged. Resolve any existing diverged_copies finding for this slug:
      ```typescript
      sqlite.prepare(
        `UPDATE project_health SET resolved_at = ? WHERE project_slug = ? AND check_type = 'diverged_copies' AND resolved_at IS NULL`
      ).run(new Date().toISOString(), slug);
      ```
   c. If HEADs differ, check ancestry using `git merge-base --is-ancestor` (can only run from a local repo that has the full history):
      - Find the local copy path (from healthDataMap entry where key ends in `:local`, or from DB copy record where host = "local")
      - Run: `execFile("git", ["merge-base", "--is-ancestor", remoteCopyHead, localCopyHead], { cwd: localPath, timeout: 5_000 })`
      - Exit 0: remote is ancestor of local (local is ahead, not diverged). Also check reverse direction to confirm.
      - Exit 1: not ancestor. Check reverse (`localHead` ancestor of `remoteHead`). If that also returns exit 1, copies are truly diverged.
      - Exit 128: unknown commit (shallow clone or force-push scenario). Treat as diverged.
   d. Check staleness: compute `remoteAge = Date.now() - new Date(remoteCopy.lastCheckedAt).getTime()`. If > 600_000 ms (10 minutes, 2 scan cycles), mark as stale.
   e. If diverged: call `upsertHealthFinding(db, sqlite, { projectSlug: slug, checkType: "diverged_copies", severity: isStale ? "warning" : "critical", detail: "...", metadata: { localHead, remoteHead, stale } })`
   f. If not diverged (one is ancestor of the other): resolve diverged_copies finding as in step (b)

**Stage 3: Event emission**

After both stages:
1. Emit `eventBus.emit("mc:event", { type: "health:changed", id: "all" })` -- always emit after a scan cycle so SSE consumers can refresh. Simple and avoids complex diffing.
2. If any diverged_copies findings were upserted (not resolved) in Stage 2, also emit: `eventBus.emit("mc:event", { type: "copy:diverged", id: slug })`

**1c. Call `runPostScanHealthPhase` from `scanAllProjects`:**

At the end of `scanAllProjects`, after the existing scan loop and before the `scan:complete` event emission:
```typescript
// Post-scan health phase
if (sqlite) {
  const healthData = getCollectedHealthData();
  try {
    await runPostScanHealthPhase(healthData, db, sqlite);
  } catch (err) {
    console.error("Health phase failed:", err);
  }
}
```

Wrap in try/catch so health phase failures do not break the existing scan cycle.

**IMPORTANT anti-patterns to avoid:**
- Do NOT run health checks inline during the per-repo scan loop. Copy reconciliation needs ALL repos scanned first.
- Do NOT bypass `upsertHealthFinding` -- it preserves detectedAt timestamps.
- Do NOT update copy records on SSH failure -- this corrupts stale detection.

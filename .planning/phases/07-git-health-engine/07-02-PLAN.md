**2c. Create `collectLocalHealthData` function:**
Collects health-relevant git data for a single local repo in ONE `sh -c` invocation to minimize process count. Returns `HealthScanData | null`.

Script commands (joined with ` && `, using `===DELIM===` separators):
1. `git remote get-url origin 2>/dev/null || echo ""`
2. `git symbolic-ref --short HEAD 2>/dev/null || echo "DETACHED"`
3. `git status -sb 2>/dev/null | head -1`
4. Branch-dependent upstream check: `BRANCH=$(git symbolic-ref --short HEAD 2>/dev/null || echo ""); test -n "$BRANCH" && git config branch.$BRANCH.remote 2>/dev/null || echo ""`
5. `git rev-list @{u}..HEAD --count 2>/dev/null || echo "-1"`
6. `git rev-list HEAD..@{u} --count 2>/dev/null || echo "-1"`
7. `git rev-parse HEAD 2>/dev/null || echo ""`

Parse the delimited output into HealthScanData fields:
- `remoteUrl`: parts[0] or null if empty
- `hasRemote`: remoteUrl is non-null and non-empty
- `isDetachedHead`: parts[1] === "DETACHED"
- `branch`: if detached then "HEAD" else parts[1]
- `upstreamGone`: parts[2] includes "[gone]"
- `hasUpstream`: parts[3] is non-empty
- `unpushedCount`: parseInt(parts[4]), use 0 if < 0
- `unpulledCount`: parseInt(parts[5]), use 0 if < 0
- `headCommit`: parts[6]
- `dirty`: pass from existing scan result
- `isPublic`: pass from cached value (null if unknown)

Signature: `async function collectLocalHealthData(projectPath: string, slug: string, dirty: boolean, cachedIsPublic: boolean | null): Promise<HealthScanData | null>`

Use `execFile("sh", ["-c", script], { cwd: projectPath, timeout: EXEC_TIMEOUT })`.

**2d. Extend SSH batch script in `scanRemoteProject`:**
Add 5 new sections to the existing SSH batch script, AFTER the existing ===GSD=== section:

```
echo "===STATUS_SB==="

git rev-list @{u}..HEAD --count 2>/dev/null || echo "-1"
echo "===REVLIST_DOWN==="
git rev-list HEAD..@{u} --count 2>/dev/null || echo "-1"
echo "===HEAD_HASH==="
git rev-parse HEAD 2>/dev/null || echo ""
```

Update `scanRemoteProject` to also return a `healthData: HealthScanData | null` field. Extend the return type. Parse the new sections into HealthScanData fields using the same logic as local. Store results in a `Map<string, HealthScanData>` that gets returned from `scanAllProjects` for the post-scan health phase.

**2f. Integrate `isPublic` cache:**
Before scanning, load existing copies from DB via `getCopiesByProject`. Extract `isPublic` from the stored copy. If `isPublic` is null after scan, attempt `gh api repos/{owner}/{repo} --jq .private` (extract owner/repo from remoteUrl). Cache result via `upsertCopy`. If `gh api` fails, leave `isPublic` as null (treat as private).

Create helper: `async function fetchIsPublic(remoteUrl: string): Promise<boolean | null>`. Parse owner/repo from normalized URL (split on `/`, take last two segments after the host). Call `execFile("gh", ["api", `repos/${owner}/${repo}`, "--jq", ".private"], { timeout: GH_TIMEOUT })`. If stdout is "false", return true (public). If "true", return false (private). On error, return null.

**2g. Update `scanAllProjects` to use p-limit and collect health data:**

- GitHub-only projects: no health data collected, no copy upsert (they have no physical clone)

<objective>
Build the insight generator service with 4 pattern detectors and wire into the daemon schedule.

Purpose: This is the core intelligence engine for Phase 37. Four rule-based pattern detectors (stale captures per D-04, activity gaps per D-05, session patterns per D-06, cross-project patterns per D-07) analyze MC data and create insights in the insights table. The daemon runs this on a schedule -- insights are never generated on API request. PROACT-03 and PROACT-04 use pure SQL aggregation (no LLM). PROACT-05 uses FTS5 term overlap with optional LM Studio summary.

Output: insight-generator.ts service, extended daemon, parameterized stale capture query, comprehensive tests.

       ```typescript
       const STALE_CAPTURE_DAYS = 7;          // D-04: 7 days without project assignment
       const MIN_CAPTURES_FOR_GAP = 3;         // D-05: minimum captures to detect gap
       const MIN_DAYS_NO_COMMIT = 7;           // D-05: days without commit for gap
       const MIN_SESSIONS_FOR_PATTERN = 10;    // D-06 / Research Pitfall 6: minimum sessions
       const MIN_SHARED_TERMS = 3;             // D-07: minimum overlapping terms
       const MAX_CROSS_PROJECT_PAIRS = 5;      // Research Pitfall 4: cap LLM calls
       ```

    3. **`generateStaleCaptureInsights(db)`** (PROACT-02, D-04):
       - Count captures older than 7d without project assignment and not archived
       - Use SQL: `SELECT count(*) FROM captures WHERE created_at < ? AND project_id IS NULL AND status != 'archived'`
       - The `?` is epoch seconds for 7 days ago
       - If count === 0, return early
       - Compute content hash: `computeContentHash(\`stale_capture:${count}:${todayIso}\`)` where todayIso is `new Date().toISOString().slice(0, 10)`
       - Call `createInsight(db, { type: "stale_capture", title: \`${count} captures need triage\`, body: \`You have ${count} capture(s) older than 7 days...\`, contentHash })`
       - Return count of insights created (0 or 1)

    4. **`detectActivityGaps(db)`** (PROACT-03, D-05):
       - Query captures in last 7 days grouped by projectId: `SELECT project_id, count(*) as cnt FROM captures WHERE created_at >= ? AND project_id IS NOT NULL GROUP BY project_id`
       - For each project with >= MIN_CAPTURES_FOR_GAP captures:
         - Query latest commit: `SELECT author_date FROM commits WHERE project_slug = ? ORDER BY author_date DESC LIMIT 1`
         - Parse authorDate (ISO string), compute daysSinceCommit
         - If daysSinceCommit >= MIN_DAYS_NO_COMMIT:
           - Content hash: `computeContentHash(\`activity_gap:${projectId}:${captureCount}:${daysSinceCommit}:${todayIso}\`)`
           - Create insight with type "activity_gap", projectSlug set, title per D-05 pattern
       - Return count of insights created

    5. **`detectSessionPatterns(db)`** (PROACT-04, D-06):
       - Query completed sessions with endedAt: `SELECT started_at, ended_at FROM sessions WHERE status = 'completed' AND ended_at IS NOT NULL ORDER BY started_at DESC LIMIT 50`
       - If fewer than MIN_SESSIONS_FOR_PATTERN, return 0
       - Compute hour distribution: Map<hour, { count, totalDurationMin }>
       - Find peak hour (most sessions) and its avg duration
       - Content hash: `computeContentHash(\`session_pattern:${peakHour}:${avgDuration}:${todayIso}\`)`
       - Create insight: "Most productive sessions start after {hourStr} and last {avgDuration}min"
       - Format hour as "10am" / "2pm" / "12pm" / "12am" style
       - Return 0 or 1

    6. **`detectCrossProjectPatterns(db)`** (PROACT-05, D-07):
       - Query recent captures (last 30 days) grouped by project, extract significant terms
       - Also query projectKnowledge content for each project
       - Build term frequency map per project: split content into words, filter stop words (a, the, is, etc.), lowercase, keep words with length >= 4
       - For each project pair, compute shared terms (intersection where both have >= 2 occurrences)
       - If shared terms >= MIN_SHARED_TERMS:
         - Content hash: `computeContentHash(\`cross_project:${slugA}:${slugB}:${sharedTerms.sort().join(",")}:${todayIso}\`)`
         - Create insight with metadata JSON containing `{ projects: [slugA, slugB], sharedTerms, evidence: [...] }`
         - Title: "{slugA} and {slugB} share patterns: {top3terms}"
         - Body: cite specific evidence per D-07 ("both reference {term}")
       - Limit to MAX_CROSS_PROJECT_PAIRS insights per cycle
       - This is pure rule-based, no LLM needed (LM Studio summary is optional enhancement, skip for v1)
       - Return count created

    7. **`generateAllInsights(db)`** — orchestrator function:
       - Calls all 4 detectors sequentially
       - Logs total insights created
       - Does NOT use LM Studio lock (rule-based, fast)

    8. **Write comprehensive tests** in `packages/api/src/__tests__/services/insight-generator.test.ts`:

       - Test detectActivityGaps: <3 captures, verify no insight

       - Test detectCrossProjectPatterns: seed overlapping captures for 2 projects, verify cross_project insight
       - Test detectCrossProjectPatterns: no overlap, verify no insight
       - Test generateAllInsights calls all detectors

**Design docs route** — Create `packages/api/src/routes/design-docs.ts`:
- GET `/` scans `~/.gstack/projects/` directory
- For each subdirectory, check for `designs/` subfolder
- Read each `.md` file in designs/, extract title from first `# ` heading or filename
- Map directory name to project name (e.g., `quartermint-gstackapp` -> `gstackapp`)
- Return array sorted by file modification time (most recent first) using `fs.statSync().mtime`
- Path safety: resolve all paths, verify under `~/.gstack/` before reading
- Empty state: return empty array `[]` if directory doesn't exist

**Worklog route** — Create `packages/api/src/routes/worklog.ts`:
- GET `/carryover` reads `~/.claude/logs/worklog.md`
- Parse with regex: split by session headers matching `/^\*\*Session\s+(\d{4}-\d{2}-\d{2})\s*--\s*(\w[\w-]*):.*\*\*/gm`
- Within each session block, find `### Carryover` section and extract bullet items (`/^[-*]\s+(.+)$/gm`)
- Compute staleness using `differenceInDays(new Date(), parseISO(loggedDate))`: < 3 = 'recent', 3-7 = 'aging', > 7 = 'stale'
- Return array sorted by date descending
- If worklog file doesn't exist, return empty array

Export `parseWorklogCarryover` and `computeStaleness` as named exports for testability.

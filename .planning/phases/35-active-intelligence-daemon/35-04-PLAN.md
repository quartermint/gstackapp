         ```typescript
         const toolCallSchema = z.discriminatedUnion("tool", [

           z.object({ tool: z.literal("project_lookup"), args: z.object({ slug: z.string() }) }),
         ```

    2. Create `packages/api/src/services/intelligence-daemon.ts`:

       The daemon orchestrator that ties all intelligence services together and registers with server lifecycle.

       ```typescript
       export interface IntelligenceDaemonConfig {
         digestCron?: string;               // Default: '0 6 * * *'
         narrativeRefreshIntervalMs?: number; // Default: 30 * 60_000 (30 min)
         cacheCleanupIntervalMs?: number;    // Default: 60 * 60_000 (1 hour)
         initialNarrativeCount?: number;     // Default: 5 (most active projects)
       }
       ```

       - `startIntelligenceDaemon(db, config?)`: Orchestrator function:
         a. Schedule daily digest via `scheduleDigestGeneration(db, config.digestCron)`
         b. Start narrative refresh timer (setInterval, 30min default):
            - Query top N projects by recent activity (lastScannedAt DESC)
            - For each: check if narrative cache is stale, regenerate if needed
         c. Start cache cleanup timer (setInterval, 1h): calls `purgeExpiredCache(db)`
         d. Run initial generation pass (setTimeout 5s, per RESEARCH.md Pitfall 6):
            - Query 5 most recently active projects
            - Regenerate narratives for each (sequential, not parallel, to avoid LM Studio overload)
            - Generate initial digest if cache is empty
         e. Returns `{ stop: () => void }` that calls digestJob.stop(), clearInterval(narrativeTimer), clearInterval(cleanupTimer)

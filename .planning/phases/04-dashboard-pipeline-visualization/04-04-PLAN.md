<objective>
Build the PR feed, PR detail view, finding cards, and feedback UI -- the data browsing and interaction layer of the dashboard.

Purpose: Users need to see all their PR reviews and drill into findings. The feed gives an at-a-glance view of review activity, the detail view lets users inspect individual findings, and the feedback UI enables signal quality improvement. This is the primary interaction surface alongside the pipeline hero.

Output: Working PR feed below the pipeline hero, expandable PR detail with stage-grouped findings, and thumbs up/down feedback on each finding.
</objective>

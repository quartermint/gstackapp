# Phase 32 Plan 01: Vector Infrastructure + Hybrid Search Pipeline

## Objective

Install sqlite-vec, create content-addressable embedding storage, build the hybrid BM25 + vector search pipeline with Reciprocal Rank Fusion, and index CLAUDE.md knowledge content in the unified search. LM Studio embedding endpoint provides vectors; graceful degradation when unavailable.

## Context

- `packages/api/src/db/queries/search.ts` — Current FTS5 search (extend, not replace)
- `packages/api/src/services/lm-studio.ts` — Health probe (extend for embeddings)
- `packages/api/src/db/schema.ts` — Drizzle schema
- `packages/api/src/db/index.ts` — Database setup
- `packages/api/src/routes/search.ts` — Search route
- `packages/api/src/db/queries/knowledge.ts` — Knowledge LIKE search (to be unified)
- `.planning/phases/32-hybrid-search-intelligence/32-CONTEXT.md` — Phase decisions

## Tasks

### Task 1: sqlite-vec installation + embedding tables migration
type="auto"

**Do:**
1. Install `sqlite-vec` npm package
2. Load sqlite-vec extension in `db/index.ts` after database creation
3. Create migration `0011_vector_search.sql`:
   - `embeddings` table: `content_hash TEXT PRIMARY KEY, source_type TEXT, source_id TEXT, embedding BLOB, model TEXT, dimensions INTEGER, created_at TEXT`
   - `vec_search` virtual table via `CREATE VIRTUAL TABLE vec_search USING vec0(content_hash TEXT, embedding float[768])` (768 dims for common embedding models)
   - Index on `embeddings(source_type, source_id)`
4. Add embeddings table to Drizzle schema
5. Write test: migration runs, vec_search table exists, can insert/query vector

**Done when:**
- `sqlite-vec` loads without error
- Embedding tables created via migration
- Vector insert + KNN query works in test

### Task 2: LM Studio embedding client + content-addressable storage
type="auto"

**Do:**
1. Create `packages/api/src/services/embedding.ts`:
   - `generateEmbedding(text: string): Promise<number[] | null>` — calls LM Studio `/v1/embeddings` endpoint
   - `computeContentHash(text: string): string` — SHA-256 of normalized content
   - `isEmbeddingAvailable(): boolean` — checks LM Studio health status
   - Uses same LM Studio URL from config (extends existing lm-studio.ts pattern)
   - Returns null when LM Studio unavailable (graceful degradation)
2. Create `packages/api/src/db/queries/embeddings.ts`:
   - `upsertEmbedding(sqlite, { contentHash, sourceType, sourceId, embedding, model, dimensions })` — INSERT OR IGNORE (content-addressable = same hash = same embedding)
   - `getEmbedding(sqlite, contentHash)` — lookup by hash
   - `searchByVector(sqlite, queryVector, limit)` — two-step: vec_search KNN first, then enrich with embeddings metadata
   - `deleteEmbedding(sqlite, contentHash)` — cleanup
3. Write tests for embedding client (mock fetch) and embedding queries (real sqlite-vec)

**Done when:**
- Embedding client calls LM Studio and returns vectors
- Content-hash dedup prevents duplicate embeddings
- Vector KNN search returns ranked results
- Graceful null return when LM Studio down

### Task 3: Hybrid search with Reciprocal Rank Fusion
type="auto"

**Do:**
1. Create `packages/api/src/services/hybrid-search.ts`:
   - `hybridSearch(sqlite, query, options)` — orchestrates BM25 + vector search
   - BM25 path: reuse existing `searchUnified()` from search.ts
   - Vector path: embed query via `generateEmbedding()`, then `searchByVector()`
   - RRF fusion: `score = sum(weight / (k + rank + 1))`, k=60
   - Original query BM25 gets 2x weight, vector gets 1x weight
   - Falls back to BM25-only when embeddings unavailable
   - Returns `HybridSearchResult[]` with combined scores
2. Update `packages/api/src/routes/search.ts`:
   - Replace `processSearchQuery` + `searchUnified` with `hybridSearch`
   - Add `vectorScore` and `bm25Score` to response (for debugging/tuning)
   - Keep existing filter support (sourceType, projectSlug, date ranges)
3. Update shared schemas for new response fields
4. Write tests: RRF scoring math, fallback behavior, end-to-end route test

**Done when:**
- Search route returns hybrid-fused results when embeddings available
- Falls back to BM25-only when LM Studio unavailable
- Response includes individual scores for tuning visibility
- RRF math verified by unit test

### Task 4: Index CLAUDE.md knowledge in unified search + backfill embeddings
type="auto"

**Do:**
1. Extend `indexCapture`, `indexCommit`, `indexProject` in search.ts to also generate embeddings async (queueMicrotask pattern from enrichment.ts)
2. Add `indexKnowledge(sqlite, { projectSlug, content })` to search.ts — indexes CLAUDE.md content in FTS5 search_index with source_type='knowledge'
3. Hook knowledge upsert (`upsertKnowledge` in knowledge.ts) to also call `indexKnowledge`
4. Create `packages/api/src/services/embedding-backfill.ts`:
   - `backfillEmbeddings(sqlite, db)` — scans search_index for entries without embeddings, generates them
   - Respects rate limiting (process N items per cycle)
   - SSE progress event for dashboard visibility
5. Write tests: knowledge appears in search results, backfill processes existing content

**Done when:**
- CLAUDE.md content searchable via unified FTS5 search
- New content gets embeddings queued async
- Backfill can process existing content
- Knowledge search no longer needs separate LIKE query path

## Layout Contract

### Session Tab Bar
- Position: Top of main content area, below sidebar header alignment
- Height: 40px
- Background: `--color-surface`
- Bottom border: 1px `--color-border`
- Tab spacing: 0px (flush tabs with 1px border between)
- Tab min-width: 120px, max-width: 200px
- Active tab: 2px bottom border in `--color-accent`, text in `--color-accent`
- Inactive tab: text in `--color-text-muted`, hover bg `--color-surface-hover`
- Close button: 16x16px X icon, visible on hover only, `--color-text-muted`
- New tab button: "+" at right end, same height, `--color-text-muted`

### Ideation View
- Layout: Two-column -- left (60%) ideation pipeline + conversation, right (40%) artifact panel
- Ideation input: Full-width textarea at top, min-height 80px, max-height 200px
- Pipeline visualization: Horizontal 4-node topology below input, same visual pattern as PipelineHero
- Conversation stream: Below pipeline, scrollable, full remaining height
- Artifact panel: Right side, scrollable list of `ArtifactCard` components
- When no artifacts: Right panel collapses, conversation takes 100% width

### Autonomous Execution View
- Layout: Two-column -- left (50%) vertical phase pipeline, right (50%) live commit stream
- Phase pipeline: Vertical flow (top to bottom), nodes connected by vertical connectors
- PhaseNode dimensions: Full-width (minus padding), height 64px (compact horizontal card, not tall like StageNode)
- Commit stream: Scrollable feed, auto-scrolls to bottom on new commits
- Agent spawn indicators: Inline with active PhaseNode, small 24x24px avatars
- Decision gate overlay: When blocking decision needed, amber-bordered card overlays the active PhaseNode area

### Decision Gate Cards
- Position: In sidebar `DecisionQueue` section
- Card size: Full sidebar width minus 8px padding each side
- Background: `--color-surface`
- Border: 1px `--color-border`, 2px left accent border
- Blocking gates: Left accent `#FFB020` (warning amber)
- Non-blocking gates: Left accent `--color-border-focus`
- Content: Title (13px Geist 500), description (13px Geist 400), option buttons
- Option buttons: Small, text-only, `--color-accent` for recommended option

### Repo Scaffold Form
- Position: Modal overlay (centered, max-width 560px)
- Background: `--color-surface`
- Border: 1px `--color-border`
- Border radius: `--radius-lg` (12px)
- Sections: Name input, description textarea, stack selector (radio group), pre-populated fields from ideation
- Primary action: "Scaffold Repository" button in `--color-accent`

## Interaction Contracts

### Ideation Flow
1. User types idea in `IdeationInput` textarea, presses "Start Ideation" (accent CTA)
2. Pipeline begins at office-hours stage -- node transitions from PENDING (20% opacity) to RUNNING (pulse glow, `#FF8B3E`)
3. SSE streams skill execution output into conversation area below pipeline
4. When office-hours completes: node transitions to COMPLETE (100% opacity, `#2EDB87` badge), artifact card appears in right panel
5. Next stage (CEO review) auto-starts -- same PENDING > RUNNING > COMPLETE cycle
6. User can click any completed stage node to view its artifact in the artifact panel
7. After all 4 stages complete: "Launch Execution" accent CTA appears below pipeline

### Autonomous GSD Execution
1. User clicks "Launch Execution" -- autonomous pipeline view replaces ideation view
2. Vertical phase pipeline renders all phases as PENDING nodes
3. SSE streams phase progress: PENDING > RUNNING > COMPLETE per phase
4. Running phase shows pulse glow (`#36C9FF`), agent spawn indicators appear
5. Commits stream in real-time in right panel as they happen
6. If decision gate triggered: `DecisionGateCard` appears in sidebar queue with amber badge count
7. Blocking gates: Pipeline pauses at that phase, PhaseNode shows amber border + "Waiting for input" label
8. Non-blocking gates: Pipeline continues, card remains in queue for user to address
9. User responds to decision gate: card dismisses, pipeline resumes if was blocking
10. On completion: `ExecutionSummary` replaces pipeline view

### Multi-Tab Sessions
1. Each new ideation or project session opens as a new tab in `SessionTabBar`
2. Status dot on tab: cyan pulse while AI is thinking, amber while waiting for user input, muted when idle
3. Switching tabs preserves full conversation state (React state + tanstack-query cache per session)
4. Close tab: If session has unsaved ideation, show confirmation ("Discard session?" with destructive styling)
5. Tab overflow: Horizontal scroll with fade gradient at right edge

### Repo Scaffolding
1. User clicks "Scaffold Repo" from completed ideation summary
2. Modal opens with form pre-populated from ideation context (name, description, stack)
3. User confirms or adjusts, clicks "Scaffold Repository"
4. Loading state: button shows spinner, disabled
5. Success: Modal closes, new session tab opens for the scaffolded project
6. Error: Inline error message below form in error semantic color

## State Mapping

| Requirement | States Required |
|-------------|-----------------|
| IDEA-01 (Office-hours brainstorm) | Empty (no idea), Input (typing), Executing (SSE stream), Complete (artifact ready), Error (skill failed) |
| IDEA-02 (Chained ideation pipeline) | Per-stage: Pending, Running, Complete, Error. Pipeline: NotStarted, InProgress, AllComplete |
| IDEA-03 (View/compare artifacts) | Loading, Loaded (list), Selected (detail view), Empty (no artifacts) |
| IDEA-04 (Repo scaffolding) | Form (editing), Submitting (loading), Success (redirect), Error (inline message) |
| AUTO-01 (One-click execution) | Ready (CTA visible), Launching (loading), Running (pipeline viz), Complete (summary), Failed (error + retry) |
| AUTO-02 (Real-time visualization) | Per-phase: Pending, Running, Complete, Failed, Blocked. Commit stream: streaming, paused |
| AUTO-03 (Decision gates) | Queue empty, Queue has items (badge count), Blocking gate active, Non-blocking gate queued |
| AUTO-04 (Smart discuss phase) | Same as decision gates -- carried forward context shown as read-only artifact reference |
| SESS-02 (Multi-tab sessions) | No tabs, Single tab, Multiple tabs, Tab overflow (scroll), Tab closing (confirmation) |

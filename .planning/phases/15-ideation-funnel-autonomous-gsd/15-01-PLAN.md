     Per D-07: each skill stage reads prior artifacts as context.

   - `buildIdeationPrompt(stage: string, priorContext: string, userIdea: string): string` — Assembles the full prompt: `{skillPrompt}\n\n{priorContext}\n\nUser's idea: {userIdea}\n\nExecute the ${stage} analysis.`

   - Per research Pitfall 4: For repo-less sessions (D-08), the agent loop will be called with `projectPath: undefined`. The skill prompt content IS the system prompt — no bash preamble execution needed because we're injecting the prompt into the agent loop, not running the skill as a CLI command.

2. **Create packages/api/src/ideation/orchestrator.ts** (per D-05, D-06):

   - `interface IdeationPipeline { id: string; sessionId: string; agentSessionId?: string; userIdea: string; stages: Map<string, 'pending' | 'running' | 'complete' | 'error'>; artifacts: Map<string, string>; status: 'running' | 'complete' | 'failed' | 'paused' }`

   - `async function* runIdeationPipeline(pipeline: IdeationPipeline): AsyncGenerator<IdeationSSEEvent>` — Iterates through IDEATION_STAGES sequentially:
     a. Yields `{ type: 'ideation:stage:start', stage }` and updates DB ideationSessions.currentStage
     b. Builds prompt via `buildIdeationPrompt(stage, buildCumulativeContext(pipeline.artifacts), pipeline.userIdea)`
     c. Calls `runAgentLoop({ prompt, maxTurns: 50, maxBudgetUsd: 3.0 })` — per D-01, per D-08 no projectPath for repo-less sessions
     d. For each agent event, yields `{ type: 'ideation:stage:event', stage, event }` — forwards the raw AgentSSEEvent
     e. On completion, calls `detectNewArtifact(stage)` to check ~/.gstack/projects/ for new files. If found, inserts ideationArtifacts row and yields `{ type: 'ideation:stage:artifact', stage, path }`
     f. Yields `{ type: 'ideation:stage:complete', stage }`
     g. On error, yields `{ type: 'ideation:stage:error', stage, error }` and breaks
     h. After all stages, yields `{ type: 'ideation:pipeline:complete' }` and updates DB status to 'complete'

   - `async function detectNewArtifact(stage: string): Promise<string | null>` — Scans ~/.gstack/projects/ for files modified in the last 5 minutes. Returns the most recent .md file path, or null. This is a best-effort heuristic — skills write design docs to this directory.

   - Export `IdeationSSEEvent` type union covering all stage event types.

   - **GET /:sessionId** — Returns ideation session state (status, currentStage, artifacts list).

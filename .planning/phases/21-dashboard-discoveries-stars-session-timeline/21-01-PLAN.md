<!-- API contracts for discoveries and stars. Executor should use these directly. -->

Discovery API: GET /api/discoveries?status=found&limit=100
Response: { discoveries: Array<{ id, path, host, status, remoteUrl, name, lastCommitAt, discoveredAt, updatedAt }> }

Discovery actions:
- PATCH /api/discoveries/:id  body: { status: "tracked" } → promotes to project
- PATCH /api/discoveries/:id  body: { status: "dismissed" } → permanently hides

Star API: GET /api/stars?limit=200
Response: { stars: Array<{ githubId, fullName, description, language, topics, htmlUrl, intent, aiConfidence, userOverride, starredAt, ... }>, total: number }

Star intent override: PATCH /api/stars/:githubId/intent  body: { intent: "reference"|"tool"|"try"|"inspiration" }

SSE event types to listen for:
- discovery:found → refetch discoveries
- discovery:promoted → refetch discoveries
- discovery:dismissed → refetch discoveries
- star:synced → refetch stars
- star:categorized → refetch stars

Existing hook pattern (use-convergence.ts):
- useState + useEffect + fetchCounter pattern
- raw fetch() (not Hono client) since discovery/star routes are factory-created sub-routers
- useCallback for refetch

Existing popover pattern (sessions-indicator.tsx):
- useRef for panelRef, click-outside via mousedown listener with setTimeout(0) delay
- Escape key listener
- Absolute positioning with z-50
- Open/close via useState toggle

Theme tokens from app.css:
- --color-terracotta (#d4713a), --color-sage (#6b8f71), --color-gold-status (#c49b2a)
- bg-surface, bg-surface-dark, bg-surface-warm, text-text-muted, etc.
- section-divider class for divider lines

  <name>Graph Data Transformation</name>
  <files>packages/web/src/lib/graph-data.ts, packages/web/src/__tests__/lib/graph-data.test.ts</files>
  <behavior>
    buildGraphData:
    - Given empty array -> returns { nodes: [], edges: [] }
    - Given 3 projects with no dependencies -> returns 3 nodes, 0 edges
    - Given project A depends on B -> returns edge { source: "a", target: "b" }
    - Given project A depends on unknown slug "z" -> edge is omitted (dangling reference filtered)
    - Node.id = project.slug, Node.name = project.name, Node.host = project.host, Node.riskLevel = project.riskLevel
    - Node.dependencyCount = project.dependsOn.length (counts ALL declared deps, not just valid ones)

    getHighlightChain:
    - Given slug with no dependencies and nothing depends on it -> returns Set containing only that slug
    - Given A depends on B -> clicking A highlights {A, B} (upstream traversal)
    - Given A depends on B -> clicking B highlights {A, B} (downstream traversal)
    - Given chain A -> B -> C -> clicking A highlights {A, B, C} (transitive upstream)
    - Given chain A -> B -> C -> clicking C highlights {A, B, C} (transitive downstream)
    - Given diamond A -> B, A -> C, B -> D, C -> D -> clicking A highlights {A, B, C, D}
    - Given cycle A -> B -> A -> returns {A, B} without infinite loop
    - Given slug not in project list -> returns Set containing only that slug
  </behavior>
  <implementation>
    Create packages/web/src/lib/graph-data.ts with:
    1. GraphNode interface extending d3-force's SimulationNodeDatum shape (but without importing d3-force -- use structural typing so this module stays in the main bundle)
    2. GraphEdge interface with source/target strings
    3. buildGraphData(projects: ProjectItem[]): { nodes: GraphNode[]; edges: GraphEdge[] }
    4. getHighlightChain(slug: string, projects: ProjectItem[]): Set<string>

    Key implementation notes:
    - Do NOT import d3-force in this file. GraphNode uses plain properties (id, x?, y?, fx?, fy?, vx?, vy?, index?) that are structurally compatible with SimulationNodeDatum. This keeps graph-data.ts out of the d3-force chunk.
    - Filter edges where target slug does not exist in the project list (per research Pitfall 3).
    - getHighlightChain builds adjacency maps once, then does BFS/DFS both upstream and downstream.
    - Cycle safety via visited set (already in the highlight set check).
  </implementation>
</feature>

# Plan 33-01: Few-Shot Categorization + Multi-Pass Extraction + Grounding

## Objective

Evolve the single-shot Gemini categorization pipeline into a multi-signal intelligence engine: few-shot examples from user corrections, multi-pass extraction (project_ref, action_item, idea, link, question), post-hoc grounding that highlights trigger words, confidence calibration from feedback, LLM fallback via LM Studio, and startup validation of few-shot examples.

## Context

- @packages/api/src/services/ai-categorizer.ts - Current single-shot Gemini categorizer
- @packages/api/src/services/enrichment.ts - Current enrichment pipeline
- @packages/api/src/db/schema.ts - Database schema
- @packages/shared/src/schemas/capture.ts - Zod schemas
- @.planning/phases/33-capture-intelligence-engine/33-CONTEXT.md - Phase context

## Tasks

### Task 1: Few-shot examples table + extraction types schema
type="auto"

Add database migration for `few_shot_examples` table and `capture_extractions` table. Add new columns to captures table for sourceType and groundingData.

**Done criteria:**
- Migration 0011 creates few_shot_examples table (id, captureContent, projectSlug, extractionType, isCorrection, createdAt)
- Migration 0011 creates capture_extractions table (id, captureId, extractionType, content, confidence, groundingJson, createdAt)
- Migration 0011 adds source_type column to captures (default 'manual')
- Drizzle schema updated with new tables
- Shared Zod schemas updated for extraction types enum

### Task 2: Enhanced AI categorizer with few-shot examples + LLM fallback
type="auto"

Rewrite ai-categorizer to use few-shot examples from DB, support LM Studio fallback when Gemini is unavailable, and return multi-pass extraction results.

**Done criteria:**
- categorizeCapture accepts few-shot examples and returns enhanced result with extractions
- LM Studio fallback when Gemini unavailable (OpenAI-compatible API)
- Multi-pass extraction: project_ref, action_item, idea, link, question
- Prompt includes few-shot examples for the capture's domain
- Tests cover few-shot prompt construction, LLM fallback, extraction types

### Task 3: Post-hoc grounding engine
type="auto"

Implement deterministic grounding that aligns extraction results back to source text, highlighting which words triggered each extraction.

**Done criteria:**
- grounding.ts service with alignExtractions function
- Cascade: MATCH_EXACT -> MATCH_LESSER -> MATCH_FUZZY -> ungrounded
- Returns character offsets for highlighted spans
- Tests cover all grounding tiers and edge cases

### Task 4: User correction API + confidence calibration
type="auto"

Add PATCH endpoint for capture project reassignment that stores corrections as few-shot examples and tracks correction rates per project for confidence calibration.

**Done criteria:**
- PATCH /api/captures/:id/reassign endpoint accepts new projectSlug
- Correction stored as new few-shot example automatically
- Correction rate tracked per project pair (predicted vs actual)
- GET /api/captures/correction-stats returns calibration data
- Startup validation checks few-shot examples still produce correct results

### Task 5: Enrichment pipeline integration + tests
type="auto"

Wire the enhanced categorizer, extraction, and grounding into the enrichment pipeline. Update enrichCapture to use few-shot examples, store extractions, and compute grounding.

**Done criteria:**
- enrichCapture uses few-shot examples from DB
- Extractions stored in capture_extractions table
- Grounding data stored on each extraction
- SSE events emitted for extraction updates
- Integration tests cover full pipeline
- All existing tests still pass

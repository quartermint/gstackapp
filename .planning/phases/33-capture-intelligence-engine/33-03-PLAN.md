    1. Create `packages/api/src/services/tweet-fetcher.ts` (per D-10):
       - Import: `pLimit` from "p-limit", `eventBus` from "./event-bus.js", `extractLinkMetadata` from "./link-extractor.js"
       - Define CRAWL4AI_URL default: "http://100.x.x.x:11235"
       - Define TWEET_URL_PATTERN: `/^https?:\/\/(twitter\.com|x\.com)\//`
       - Export interface `TweetContent { url: string; content: string | null; author: string | null; threadContext: string | null; fetchedVia: "crawl4ai" | "og_scraper" | "failed" }`
       - Export `async function isCrawl4AIAvailable(baseUrl = CRAWL4AI_URL): Promise<boolean>`
         - Try: `fetch(\`${baseUrl}/health\`, { signal: AbortSignal.timeout(3000) })`
         - Return true if response.ok, false otherwise
         - Catch: return false
       - Export `async function fetchTweetContent(url: string, crawl4aiUrl = CRAWL4AI_URL): Promise<TweetContent>`
         - If not TWEET_URL_PATTERN: return { url, content: null, author: null, threadContext: null, fetchedVia: "failed" }
         - Try Crawl4AI first:
           - POST to `${crawl4aiUrl}/crawl` with body: `{ urls: [url], word_count_threshold: 10, extraction_strategy: { type: "NoExtractionStrategy" } }`
           - Timeout: 30 seconds (tweets can be slow)
           - Parse response, extract text content from result
           - If result has content: extract author from URL path (`/handle/status/id`), return with fetchedVia: "crawl4ai"
         - Fallback to OG scraper:
           - Call `extractLinkMetadata(url)` (from existing link-extractor.ts)
           - Return { url, content: metadata.description, author: null, threadContext: null, fetchedVia: "og_scraper" }
         - On all failure: return with fetchedVia: "failed"
       - Export `async function batchFetchTweets(urls: string[], crawl4aiUrl = CRAWL4AI_URL, onProgress?: (done: number, total: number) => void): Promise<TweetContent[]>`
         - Use p-limit with concurrency 2 (per pitfall 4: rate limiting)
         - Process each URL with fetchTweetContent
         - Between each request: `await new Promise(r => setTimeout(r, 1000))` (1-second delay per pitfall)
         - On 429-like failures: exponential backoff (2^retryCount * 1000ms), max 3 retries
         - Emit progress: `eventBus.emit("mc:event", { type: "tweet-fetch:progress", done, total: urls.length })`
         - Call onProgress callback if provided
         - Return array of TweetContent results

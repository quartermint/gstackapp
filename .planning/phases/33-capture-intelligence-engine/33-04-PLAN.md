    1. Create `packages/api/src/services/imessage-monitor.ts` (per D-12, D-13, D-14):
       - Import: `Database` from "better-sqlite3", `eventBus` from "./event-bus.js", `nanoid` from "nanoid"
       - Import types: `DrizzleDb` from "../db/index.js"
       - Import: `createCapture` from "../db/queries/captures.js"

       - Constants:
         - `APPLE_EPOCH_OFFSET = 978307200` (seconds between 1970-01-01 and 2001-01-01)
         - `NANOSECOND_DIVISOR = 1_000_000_000`
         - `DEFAULT_POLL_INTERVAL_MS = 5 * 60 * 1000` (5 minutes)

       - Export `function convertAppleTimestamp(appleNanos: number): Date`:
         - `const unixSeconds = (appleNanos / NANOSECOND_DIVISOR) + APPLE_EPOCH_OFFSET`
         - `return new Date(unixSeconds * 1000)`

       - Export `function dateToAppleNanos(date: Date): number`:
         - `const unixSeconds = date.getTime() / 1000`
         - `return (unixSeconds - APPLE_EPOCH_OFFSET) * NANOSECOND_DIVISOR`

       - Export `function extractAttributedBodyText(attributedBody: Buffer | null): string | null`:
         - If null or empty: return null
         - Strategy: search for readable UTF-8 string content between binary plist markers
         - Try regex approach: convert to string, match `/(?<=NSString\x01.).+?(?=\x00)/s`
         - If match found: return match[0].trim()
         - Fallback: scan for sequences of printable ASCII/UTF-8 chars > 5 chars long
         - If nothing found: return null

       - Export interface `IMessageEntry { messageId: number; text: string; isFromMe: boolean; handleId: string; chatIdentifier: string; timestamp: Date; conversationPartner: string }`

       - Export `function pollNewMessages(chatDbPath: string, contactIdentifiers: string[], sinceAppleNanos: number): { messages: IMessageEntry[]; latestTimestamp: number }`:
         - Open: `new Database(chatDbPath, { readonly: true, fileMustExist: true })`
         - Set: `db.pragma("busy_timeout = 1000")` (per pitfall 6)
         - Query:
           ```sql
           SELECT
             m.ROWID as messageId,
             m.text,
             m.attributedBody,
             m.date as appleDate,
             m.is_from_me as isFromMe,
             h.id as handleId,
             c.chat_identifier as chatIdentifier
           FROM message m
           JOIN chat_message_join cmj ON cmj.message_id = m.ROWID
           JOIN chat c ON c.ROWID = cmj.chat_id
           LEFT JOIN handle h ON m.handle_id = h.ROWID
           WHERE c.chat_identifier IN (placeholder_list)
             AND m.date > ?
           ORDER BY m.date ASC
           ```
         - Generate placeholder list dynamically: `contactIdentifiers.map(() => "?").join(",")`
         - Parameters: `[...contactIdentifiers, sinceAppleNanos]`
         - For each row: check `text` first, if NULL call `extractAttributedBodyText(row.attributedBody)`
         - Skip rows where both text and attributedBody extraction fail (log warning)
         - Convert appleDate to JS Date via convertAppleTimestamp
         - Track latestTimestamp as max appleDate seen
         - Close database immediately
         - Return { messages, latestTimestamp }

       - Export interface `IMessageMonitorConfig { chatDbPath: string; contacts: string[]; pollIntervalMs: number; enabled: boolean }`

       - Export `function startIMessageMonitor(mcDb: DrizzleDb, config: IMessageMonitorConfig): ReturnType<typeof setInterval> | null`:
         - If !config.enabled or config.contacts.length === 0: log and return null
         - Initialize lastSeenTimestamp to 24 hours ago (dateToAppleNanos(new Date(Date.now() - 86400000)))
         - Define async poll function:
           - Try: call pollNewMessages(config.chatDbPath, config.contacts, lastSeenTimestamp)
           - For each new message:
             - Build capture content: if isFromMe, prefix "Ryan: "; else prefix "Bella: " (or contact name from handleId)
             - Include conversation context: `"From conversation with ${conversationPartner}"`
             - Create capture via createCapture(mcDb, { rawContent: messageText, type: "text", sourceType: "imessage", sourceMetadata: JSON.stringify({ handleId, chatIdentifier, isFromMe, timestamp }) })
             - Fire-and-forget enrichment: `queueMicrotask(() => enrichCapture(mcDb, capture.id).catch(...))`
             - Emit SSE event: `eventBus.emit("mc:event", { type: "imessage:new", count: messages.length })`
           - Update lastSeenTimestamp with latestTimestamp from poll
           - Log: `iMessage: polled ${messages.length} new message(s)`
         - Catch errors:
           - If SQLITE_CANTOPEN or EPERM: log "iMessage: Full Disk Access required. Grant in System Settings > Privacy & Security." and disable further polling (clearInterval)
           - Other errors: log and continue
         - Run poll immediately, then on interval
         - Return interval handle

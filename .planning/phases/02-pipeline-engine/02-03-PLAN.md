**Function `runStage(input: StageInput): Promise<StageOutput>`:**

1. Record `startTime = Date.now()`.
2. Create `AbortController`. Set timeout: `setTimeout(() => controller.abort(), STAGE_TIMEOUT_MS)`. Clear in finally.
3. Load stage instructions: `readFileSync(resolve(dirname(fileURLToPath(import.meta.url)), 'prompts', input.stage + '.md'), 'utf-8')`. Use `import.meta.url` with `fileURLToPath` and `dirname` for ESM compatibility.
4. Create sandbox tools: `createSandboxTools(input.clonePath)`.
5. Build user message content via `buildStageInput(input)` helper (defined below).
6. Initialize messages array: `[{ role: 'user' as const, content: userContent }]`.
7. Loop up to MAX_ITERATIONS:
   a. Call `anthropic.messages.create({ model: MODEL_MAP[input.stage], max_tokens: 4096, system: [{ type: 'text', text: stageInstructions, cache_control: { type: 'ephemeral' } }], tools: toolsWithCaching, messages }, { signal: controller.signal })`.
   b. For `tools` parameter: clone the tools array and add `cache_control: { type: 'ephemeral' as const }` to the LAST tool definition only (for prompt caching).
   c. Accumulate `totalTokens += response.usage.input_tokens + response.usage.output_tokens`.
   d. If `response.stop_reason === 'end_turn'` or `'refusal'`: break, set finalResponse.
   e. If `response.stop_reason === 'tool_use'`: extract tool_use blocks, execute each via `executeTool()`, build tool_result blocks. CRITICAL ordering: tool_result must be in a user message immediately after the assistant message. Format: `messages.push({ role: 'assistant', content: response.content }, { role: 'user', content: toolResults })`.
   f. Other stop_reason (max_tokens, etc.): break, set finalResponse.
8. Parse structured output from finalResponse using `parseStageOutput(finalResponse, input.stage)`.
9. Return StageOutput.

**IMPORTANT per RESEARCH.md anti-patterns:**
- Do NOT use `token-efficient-tools-2025-02-19` beta header (no effect on Claude 4+).
- Do NOT prefill assistant messages (returns 400 on Claude 4.6).
- Tool results MUST come first in user content array before any text.

**Helper `buildStageInput(input: StageInput): string`:**

Build a markdown-formatted string with:
- PR number, repo, head SHA, file count, total additions/deletions
- List of changed files with status and line counts
- Truncated diff patches for the most significant files (sorted by change size, 15000 char budget)
- Instruction to use read_file, list_files, search_code tools for deeper exploration

**Helper `parseStageOutput(response: Anthropic.Message | null, stage: Stage): { verdict: Verdict; summary: string; findings: Finding[] }`:**

1. If response is null: return `{ verdict: 'FLAG', summary: 'Stage did not produce a response', findings: [] }`.
2. Extract text blocks from response.content.
3. Try to find a JSON code block via regex: `/```json\s*([\s\S]*?)\s*```/`.
4. If found, parse JSON, validate verdict with VerdictSchema.parse(), validate each finding with FindingSchema.parse().
5. If JSON parsing/validation fails, fall back: return `{ verdict: 'FLAG', summary: fullText.slice(0, 500), findings: [] }`.

**Function `runStageWithRetry(input: StageInput): Promise<StageOutput>`:**

Per D-11: retry once on error, then FLAG.
```typescript
export async function runStageWithRetry(input: StageInput): Promise<StageOutput> {
  try {
    return await runStage(input)
  } catch (firstError) {
    console.warn(`[stage-runner] ${input.stage} failed, retrying:`, (firstError as Error).message)
    try {
      return await runStage(input)
    } catch (retryError) {
      return {
        verdict: 'FLAG' as const,
        summary: `Stage failed after retry: ${(retryError as Error).message}`,
        findings: [],
        tokenUsage: 0,
        durationMs: 0,
      }
    }
  }
}
```

**Function `executePipeline(input: PipelineInput): Promise<void>`:**

Full lifecycle:

1. **PIPE-09: Set RUNNING status** immediately, before anything else:
   ```typescript
   db.update(pipelineRuns)
     .set({ status: 'RUNNING', startedAt: new Date() })
     .where(eq(pipelineRuns.id, input.runId))
     .run()
   ```

2. **Initialize clonePath to null** for cleanup in finally block.

3. **Try block:**

   a. Get installation Octokit: `getInstallationOctokit(input.installationId)`.

   b. Fetch PR changed files:
   ```typescript
   const [owner, repo] = input.repoFullName.split('/')
   const { data: prFiles } = await octokit.pulls.listFiles({
     owner, repo, pull_number: input.prNumber,
   })
   ```
   Map to PrFile format: `prFiles.map(f => ({ filename: f.filename, status: f.status, additions: f.additions, deletions: f.deletions, patch: f.patch }))`

   c. **D-08/D-09: Smart filtering** -- determine which stages to run:
   ```typescript
   const ALL_STAGES: Stage[] = ['ceo', 'eng', 'design', 'qa', 'security']
   const stagesToRun = ALL_STAGES.filter(stage => shouldRunStage(stage, mappedFiles))
   ```

   d. **Clone only if needed** -- if stagesToRun.length > 0:
   ```typescript
   clonePath = await cloneRepo(input.installationId, input.repoFullName, input.headRef)
   ```

   e. **Create stage_result records** for ALL 5 stages. Stages that will run get verdict 'RUNNING'. Stages that are skipped get verdict 'SKIP':
   ```typescript
   const stageResultIds = new Map<string, string>()
   for (const stage of ALL_STAGES) {
     const stageId = nanoid()
     stageResultIds.set(stage, stageId)
     db.insert(stageResults).values({
       id: stageId,
       pipelineRunId: input.runId,
       stage,
       verdict: stagesToRun.includes(stage) ? 'RUNNING' : 'SKIP',
     }).run()
   }
   ```

   f. **PIPE-02: Fan out with Promise.allSettled:**
   ```typescript
   const results = await Promise.allSettled(
     stagesToRun.map(stage =>
       runStageWithRetry({
         stage, runId: input.runId, clonePath: clonePath!,
         prFiles: mappedFiles, repoFullName: input.repoFullName,
         prNumber: input.prNumber, headSha: input.headSha,
       })
     )
   )
   ```

   g. **Fan in: persist results.** For each stage result:
   - If fulfilled: Update stage_result with verdict, summary, tokenUsage, durationMs, completedAt. Then insert each finding with `db.insert(findings).values({ id: nanoid(), stageResultId: stageResultIds.get(stage)!, pipelineRunId: input.runId, ...finding }).run()`.
   - If rejected: Update stage_result with verdict='FLAG', error=reason.message, completedAt.

   **IMPORTANT: Use the stageResultIds map** to get the correct stage_result ID. Update by the composite key (pipelineRunId + stage) using `and(eq(stageResults.pipelineRunId, input.runId), eq(stageResults.stage, stage))`.

   h. **Set COMPLETED status:**
   ```typescript
   db.update(pipelineRuns)
     .set({ status: 'COMPLETED', completedAt: new Date() })
     .where(eq(pipelineRuns.id, input.runId))
     .run()
   ```

4. **Catch block:** Set FAILED status with completedAt.

5. **Finally block:** If clonePath is not null, call `cleanupClone(clonePath)`.

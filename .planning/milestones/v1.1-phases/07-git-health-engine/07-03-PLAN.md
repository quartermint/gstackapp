**1a. Add new imports at top of file:**
- Import `runHealthChecks`, `normalizeRemoteUrl`, `escalateDirtySeverity` from `./git-health.js`
- Import `upsertHealthFinding`, `resolveFindings`, `getActiveFindings` from `../db/queries/health.js`

- NOTE: Do NOT import `computeHealthScore` here. The 0-100 health score is computed on-demand at the Phase 8 API layer from active findings (matching the `getProjectRiskLevel()` pattern from Phase 6). This plan persists findings; the score is derived from them at query time.

**1b. Create `runPostScanHealthPhase` function:**

Signature:
```typescript
async function runPostScanHealthPhase(
  healthDataMap: Map<string, HealthScanData>,
  db: DrizzleDb,
  sqlite: Database.Database
): Promise<void>
```

The function has four stages:

**Stage 1: Per-repo health checks + finding persistence**

For each entry in `healthDataMap`:
1. Call `runHealthChecks(healthScanData)` to get findings array
2. For each finding, call `upsertHealthFinding(db, sqlite, finding)` (preserves detectedAt on re-detection)
3. Collect the `checkType` values from the findings array as `activeCheckTypes`
4. Call `resolveFindings(sqlite, slug, activeCheckTypes)` to resolve any findings that no longer apply
   - Extract the project slug from the map key (format is `${slug}:${host}` -- split on `:` and take first part)
   - IMPORTANT: For multi-copy projects, resolveFindings should only resolve per-repo check types (not `diverged_copies` -- that is handled in Stage 3)
   - Add `"diverged_copies"` to activeCheckTypes when calling resolveFindings, so it does not get auto-resolved here

**Stage 2: Dirty working tree severity escalation (HLTH-06)**

After Stage 1 persists all findings (including new dirty_working_tree at "info"):
1. Query all active dirty_working_tree findings: `getActiveFindings(db).filter(f => f.checkType === "dirty_working_tree")`
2. For each dirty finding, compute escalated severity: `escalateDirtySeverity(finding.detectedAt)`
3. If escalated severity differs from stored severity, re-upsert the finding with the new severity:
   ```typescript
   if (escalated !== finding.severity) {
     upsertHealthFinding(db, sqlite, {
       projectSlug: finding.projectSlug,
       checkType: "dirty_working_tree",
       severity: escalated,
       detail: finding.detail,
       metadata: finding.metadata,
     });
   }
   ```
   This works because `upsertHealthFinding` finds the existing active finding (same slug + checkType, not resolved) and UPDATEs severity while preserving detectedAt.

This ensures that:
- Fresh detections start at "info"
- On subsequent scan cycles, if detectedAt is 3+ days old, severity escalates to "warning"
- On subsequent scan cycles, if detectedAt is 7+ days old, severity escalates to "critical"
- The persisted severity levels mean the Phase 8 API can derive a correct health score on-demand via `computeHealthScore(activeFindings)` without any scan-time score computation

**Stage 3: Multi-copy divergence detection**

Group all scanned copies by their normalized remote URL:
1. Build a `Map<string, HealthScanData[]>` from healthDataMap, where key is `normalizeRemoteUrl(scanData.remoteUrl)` (skip entries with null remoteUrl)
2. Also load existing copy records from DB via `getCopiesByRemoteUrl` for each unique remote URL (this picks up copies from hosts that were not scanned in this cycle, e.g., Mac Mini was unreachable)
3. For each group with 2+ copies (combining fresh scan data and DB records):
   a. Compare HEAD commits between copies
   b. If all HEADs match: not diverged. Resolve any existing diverged_copies finding for this slug:
      ```typescript
      sqlite.prepare(
        `UPDATE project_health SET resolved_at = ? WHERE project_slug = ? AND check_type = 'diverged_copies' AND resolved_at IS NULL`
      ).run(new Date().toISOString(), slug);
      ```
   c. If HEADs differ, check ancestry. Extract this into a testable function:

      ```typescript
      export async function checkAncestry(
        localPath: string,
        headA: string,
        headB: string
      ): Promise<"ancestor" | "descendant" | "diverged" | "unknown">
      ```

      Implementation:
      - Run: `execFile("git", ["merge-base", "--is-ancestor", headA, headB], { cwd: localPath, timeout: 5_000 })`
      - Exit 0: headA is ancestor of headB -> return "ancestor"
      - Exit 1: not ancestor. Check reverse: `execFile("git", ["merge-base", "--is-ancestor", headB, headA], ...)`
        - Exit 0: headB is ancestor of headA -> return "descendant"
        - Exit 1: neither is ancestor -> return "diverged"
      - Exit 128 (or other error): unknown commit (shallow clone or force-push) -> return "unknown"

   d. Check staleness: compute `remoteAge = Date.now() - new Date(remoteCopy.lastCheckedAt).getTime()`. If > 600_000 ms (10 minutes, 2 scan cycles), mark as stale.
   e. If diverged or unknown: call `upsertHealthFinding(db, sqlite, { projectSlug: slug, checkType: "diverged_copies", severity: isStale ? "warning" : "critical", detail: "...", metadata: { localHead, remoteHead, stale } })`
   f. If ancestor or descendant (one is ahead of the other): resolve diverged_copies finding as in step (b)

**Stage 4: Event emission**

After all stages:
1. Emit `eventBus.emit("mc:event", { type: "health:changed", id: "all" })` -- always emit after a scan cycle so SSE consumers can refresh. Simple and avoids complex diffing.
2. If any diverged_copies findings were upserted (not resolved) in Stage 3, also emit: `eventBus.emit("mc:event", { type: "copy:diverged", id: slug })`

**1c. Call `runPostScanHealthPhase` from `scanAllProjects`:**

At the end of `scanAllProjects`, after the existing scan loop and before the `scan:complete` event emission:
```typescript
// Post-scan health phase
if (sqlite) {
  const healthData = getCollectedHealthData();
  try {
    await runPostScanHealthPhase(healthData, db, sqlite);
  } catch (err) {
    console.error("Health phase failed:", err);
  }
}
```

Wrap in try/catch so health phase failures do not break the existing scan cycle.

**IMPORTANT anti-patterns to avoid:**
- Do NOT run health checks inline during the per-repo scan loop. Copy reconciliation needs ALL repos scanned first.
- Do NOT bypass `upsertHealthFinding` -- it preserves detectedAt timestamps.
- Do NOT update copy records on SSH failure -- this corrupts stale detection.

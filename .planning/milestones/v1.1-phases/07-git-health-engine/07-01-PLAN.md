  <behavior>
    ## HealthScanData Interface
    The `HealthScanData` interface represents git state collected per-repo:
    - slug, branch, dirty, remoteUrl, hasRemote, isDetachedHead, hasUpstream, upstreamGone
    - unpushedCount, unpulledCount, headCommit, isPublic

    ## Individual Check Functions (HealthScanData -> HealthFindingInput | null)

    ### checkNoRemote (HLTH-02)
    - hasRemote=false -> returns finding with checkType "no_remote", severity "critical"
    - hasRemote=true -> returns null

    ### checkBrokenTracking (HLTH-03)
    - hasRemote=true AND not detached AND hasUpstream=false AND not gone -> returns finding with checkType "broken_tracking", severity "critical", detail mentioning branch name
    - Wait, research clarifies: broken_tracking = config exists but ref resolution fails. No upstream configured is different. Per research open question #2: if hasRemote=true but hasUpstream=false, this is "no upstream configured" not "broken tracking". The check detects when upstream WAS configured but is now broken. Since we detect this via the dependency chain (hasUpstream=false means git config returned empty = no upstream at all, vs hasUpstream=true but @{u} fails = broken), the function should:
      - hasRemote=true AND not detached AND hasUpstream=false AND not gone -> returns "broken_tracking" critical finding
      - NOTE: Per research, this covers both "never configured" and "broken" cases. The distinction between "intentionally no upstream" vs "broken upstream" is deferred.

    ### checkRemoteBranchGone (HLTH-04)
    - upstreamGone=true -> returns finding with checkType "remote_branch_gone", severity "critical"
    - upstreamGone=false -> returns null

    ### checkUnpushedCommits (HLTH-01, HLTH-07)
    - No remote, detached, no upstream, or gone -> returns null (other checks handle these)
    - unpushedCount=0 -> returns null
    - unpushedCount=3, isPublic=false -> warning
    - unpushedCount=3, isPublic=true -> critical (public escalation, HLTH-07)
    - unpushedCount=8, isPublic=false -> critical (6+ threshold)
    - unpushedCount=8, isPublic=true -> critical
    - Detail includes count and "(public repo)" when applicable

    ### checkUnpulledCommits (HLTH-05)
    - No remote, detached, no upstream, or gone -> returns null
    - unpulledCount=0 -> returns null
    - unpulledCount=5 -> warning with count in detail

    ### checkDirtyWorkingTree (HLTH-06)
    - dirty=false -> returns null
    - dirty=true -> returns finding with checkType "dirty_working_tree", severity "info"
    - Note: The check function always returns "info" for initial detection. Severity escalation is handled by `escalateDirtySeverity` (see below), which is called in the post-scan phase (Plan 03) to re-upsert with age-based severity.

    ## escalateDirtySeverity (HLTH-06 severity escalation)
    Pure function: `escalateDirtySeverity(detectedAt: string, now?: Date): HealthSeverity`
    - Computes age in days from `detectedAt` ISO string to `now` (defaults to new Date())
    - age < 3 days -> "info"
    - age >= 3 days and < 7 days -> "warning"
    - age >= 7 days -> "critical"
    - Invalid/unparseable detectedAt -> "info" (safe fallback)

    Test cases:
    - detectedAt = 1 hour ago -> "info"
    - detectedAt = 2 days ago -> "info"
    - detectedAt = 3 days ago -> "warning"
    - detectedAt = 5 days ago -> "warning"
    - detectedAt = 7 days ago -> "critical"
    - detectedAt = 30 days ago -> "critical"
    - detectedAt = "invalid" -> "info"

    ## runHealthChecks Orchestrator
    - Executes checks in dependency order per research Pattern 2
    - No remote -> returns [no_remote finding] and short-circuits (no further checks)
    - Detached HEAD -> skips upstream-dependent checks (unpushed, unpulled, broken tracking, gone)
    - Gone -> still runs remaining checks but unpushed/unpulled are skipped
    - Broken tracking -> unpushed/unpulled skipped
    - Dirty working tree -> always runs (independent of remote state)
    - Returns array of all findings

    ## normalizeRemoteUrl (COPY-01)
    - "git@github.com:owner/repo.git" -> "github.com/owner/repo"
    - "https://github.com/owner/repo.git" -> "github.com/owner/repo"
    - "git@github.com:owner/repo" -> "github.com/owner/repo"
    - "https://github.com/owner/repo" -> "github.com/owner/repo"
    - Case insensitive: "Git@GitHub.com:Owner/Repo.git" -> "github.com/owner/repo"
    - Strips trailing slash

    ## computeHealthScore (HLTH-08)
    - Empty findings array -> 100
    - Only info findings -> 100
    - Worst is warning -> 60
    - Worst is critical -> 20

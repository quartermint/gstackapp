## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> operator API | Operator submits clarification answers and approvals via POST routes |
| operator API -> Claude API | Server calls Claude to generate questions/briefs |
| state machine transitions | Status changes must be validated server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Tampering | state-machine.ts | mitigate | Validate all transitions via canTransition() before DB update; reject invalid transitions with error |
| T-18-02 | Tampering | clarify-answer route | mitigate | Zod validation on answer input (max 5000 chars); session isolation check (operator can only answer own requests) |
| T-18-03 | Elevation of privilege | approve-brief route | mitigate | Verify request.userId === user.id for operators; verify status === 'briefing' before allowing approval |
| T-18-04 | Tampering | clarifier.ts | mitigate | Clarification answers passed as structured JSON to Claude API, not interpolated into system prompt strings |
| T-18-05 | Denial of service | clarify-answer route | mitigate | Max 5 questions enforced by checking previousQA.length; auto-transitions to briefing after 5 |
| T-18-06 | Spoofing | escalate route | mitigate | Session isolation: only request owner or admin can escalate |
</threat_model>

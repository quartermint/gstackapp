## Cross-Phase Integration

### Verified Integration Points

1. **Phase 32 search → Phase 34 solutions indexed** — `indexSolution` in FTS5 `search_index` with `source_type='solution'`; `SearchSourceType` extended. Solutions searchable via `/api/search`.
2. **Phase 33 captures → Phase 37 stale detection** — `getStaleCaptures` in captures queries, consumed by `generateStaleCaptureInsights()` in insight-generator.
3. **Phase 34 solutions → Phase 35 MCP learnings** — `getRelevantSolutions` called in knowledge digest route; `cross-project-search.ts` filters solutions.
4. **Phase 35 intelligence cache → Phase 37 digest strip** — DigestStripView reads from `useDigest` hook → `/api/intelligence/digest` → `intelligence_cache` table.
5. **Phase 35 daemon → Phase 37 insight scheduling** — `intelligence-daemon.ts` schedules `generateAllInsights` on 30min interval + startup.
6. **Phase 36 device hints → Phase 33 enrichment routing** — `enrichment.ts` accepts `deviceHint` parameter, short-circuits at `confidence > 0.8`.
7. **Phase 38 chat → Phase 32 search + Phase 34 solutions** — `chat-tools.ts` includes `search_knowledge` tool invoking hybrid search.

### Integration Gaps

8. **Phase 38 searchMC → Phase 32 hybrid search (BROKEN)** — The `searchMC` chat tool in `chat-tools.ts` calls `searchUnified(sqlite, query)` (BM25-only FTS5) instead of `hybridSearch(sqlite, db, query)`. Bella's conversational search bypasses the entire Phase 32 pipeline: no vector search, no RRF fusion, no cross-encoder reranking, no query expansion, no context annotations. **Fix:** Import `hybridSearch` from `services/hybrid-search.ts` — the `createChatTools` factory already receives `dbInstance` with both `db` and `sqlite`. Affects BELLA-07.

9. **Phase 33 extractions → Phase 38 Bella extraction viewing** — Chat tools can query captures but the captures list API doesn't include extractions inline. Low severity — extraction data accessible via `/captures/:id/extractions` individual endpoint.

10. **Stale capture threshold mismatch** — `insight-generator.ts` uses 7-day threshold for `stale_capture` insights, but `getStaleCaptures` defaults to 14 days. Triage view shows more captures than the insight claims. UX inconsistency, not a functional break.

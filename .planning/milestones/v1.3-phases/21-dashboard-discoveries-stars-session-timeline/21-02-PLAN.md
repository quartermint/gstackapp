Fields: `id`, `source` ("claude-code" | "aider"), `model: string | null`, `tier`, `projectSlug: string | null`, `cwd: string`, `status: string`, `startedAt: string`, `endedAt: string | null`, `lastHeartbeatAt: string | null`.
- Export `useSessionHistory()` hook: fetches `GET /api/sessions?limit=100` (no status filter — include active + completed). Uses fetchCounter pattern (same as use-convergence.ts). Parse response as `{ sessions: SessionHistoryItem[], total: number }`.

**timeline-bar.tsx:**
- Props: `session: SessionHistoryItem`, `dayStartHour: number` (e.g., 6 for 6am), `dayEndHour: number` (e.g., 24 for midnight), `containerWidth: number`
- Computes bar position and width from startedAt/endedAt relative to day window:
  - `startMinutes = (startHour - dayStartHour) * 60 + startMinute`
  - `endMinutes` = endedAt ? computed : now (for active sessions, bar extends to current time)
  - `totalMinutes = (dayEndHour - dayStartHour) * 60`
  - `left = (startMinutes / totalMinutes) * 100` (percentage)
  - `width = ((endMinutes - startMinutes) / totalMinutes) * 100` (percentage, min 2% for visibility)
- Color by source: `claude-code` → `bg-blue-500/70` (blue), `aider` → `bg-amber-warm/70` (warm)
- Active sessions have a subtle pulse animation on the right edge (a small `animate-pulse` dot)
- Bar height: `h-3 rounded-sm`
- Tooltip on hover (via title attribute): "Session {id.slice(0,8)} — {model ?? source} — {status}"

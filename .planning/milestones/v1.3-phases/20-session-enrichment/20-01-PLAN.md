<feature>
  <name>Convergence Detection Algorithm</name>
  <files>packages/api/src/services/convergence-detector.ts, packages/api/src/__tests__/services/convergence-detector.test.ts, packages/shared/src/schemas/health.ts, packages/shared/src/index.ts</files>
  <behavior>
    Given: Sessions data from the sessions table
    When: detectConvergence(db) is called

    Positive cases:
    - 2 sessions on same project, overlapping files, 1 completed, both within 30min → returns convergence result
    - 2 sessions on same project, overlapping files, both completed, both within 30min → returns convergence result
    - 3 sessions where 2 overlap and 1 is committed → returns convergence for the 2 that overlap

    Negative cases (false positive control):
    - 2 sessions on same project, NO file overlap → returns empty
    - 2 sessions on same project, file overlap, but NEITHER committed → returns empty
    - 2 sessions on same project, file overlap, 1 committed, but outside 30min window → returns empty
    - 1 active session alone on project → returns empty
    - 2 sessions on DIFFERENT projects with file overlap → returns empty

    Edge cases:
    - Session with null/empty filesJson → excluded from convergence check
    - Session with null projectSlug → excluded
    - Session with status "abandoned" → excluded (only "active" and "completed" count)
    - Exactly at 30-minute boundary → included (window is inclusive)
  </behavior>
  <implementation>
    1. Add "convergence" to healthCheckTypeEnum in shared/src/schemas/health.ts
    2. Export updated enum from shared/src/index.ts (already exported, just the enum value changes)
    3. Create convergence-detector.ts with:
       - ConvergenceResult interface: { projectSlug: string, sessions: { id: string, status: string }[], overlappingFiles: string[], detectedAt: string }
       - detectConvergence(db: DrizzleDb, windowMinutes?: number): ConvergenceResult[]
         - Default windowMinutes = 30
         - Query all sessions where status IN ('active', 'completed') AND (endedAt IS NULL OR endedAt > NOW - windowMinutes) AND projectSlug IS NOT NULL
         - Group by projectSlug
         - For each project group with 2+ sessions:
           - Check that at least one session has status === 'completed'
           - For each pair, parse filesJson, normalize paths with normalizePath from conflict-detector.ts, compute intersection
           - If intersection is non-empty → add ConvergenceResult
         - Return all ConvergenceResults
    4. Write comprehensive tests following TDD RED-GREEN-REFACTOR
  </implementation>
</feature>

  <behavior>
    - TTLCache stores and retrieves values with expiry
    - TTLCache returns undefined for expired entries
    - TTLCache.invalidateAll clears all entries
    - scanProject returns git status (branch, dirty, dirtyFiles, lastCommit) for a valid repo
    - scanProject handles missing/non-git directories gracefully (returns null or error data)
    - scanProject reads GSD state from .planning/STATE.md if it exists
    - upsertProject creates a new project record or updates existing
    - listProjects returns all projects, optionally filtered by host
    - getProject returns single project by slug or throws NOT_FOUND
    - GET /api/projects returns list of projects with git status data
    - GET /api/projects/:slug returns single project detail
    - POST /api/projects/refresh triggers re-scan and returns 202
  </behavior>
  <action>
    **`packages/api/src/services/cache.ts`:**
    Implement per research Code Examples (TTL Cache). Generic `TTLCache<T>` class with `get(key)`, `set(key, data)`, `invalidate(key)`, `invalidateAll()`. Default TTL: 60 seconds.

    **`packages/api/src/services/project-scanner.ts`:**
    Per research Pattern 4 (Project Scanner with TTL Cache). Use `child_process.execFile` with `promisify` (NOT simple-git -- per research Open Question 2 recommendation).

    Functions:
    - `scanProject(projectPath: string)`: Run three parallel git commands: `rev-parse --abbrev-ref HEAD` (branch), `status --porcelain` (dirty files), `log -5 --format=%h|%s|%ar` (last 5 commits). Set 10-second timeout on all. Parse results into `GitScanResult` type. Return null if path doesn't exist or isn't a git repo.
    - `scanAllProjects(config: MCConfig)`: Scan all projects from config in parallel using `Promise.allSettled`. Upsert results into SQLite. Cache results in TTLCache keyed by slug.
    - `getProjectWithScanData(slug: string)`: Read from cache first. If cache miss, read from database (stale but available). Return merged project + scan data.
    - `startBackgroundPoll(config: MCConfig, intervalMs?: number)`: Start `setInterval` that calls `scanAllProjects`. Default interval: 5 minutes (300000ms). Return the timer handle for cleanup.

    Also read GSD state: Check if `{repoPath}/.planning/STATE.md` exists. If so, parse the YAML frontmatter for `status`, `stopped_at`, `progress.percent`. Include in scan result as `gsdState: { status, stoppedAt, percent } | null`.

    **`packages/api/src/db/queries/projects.ts`:**
    - `upsertProject(data)`: Insert or update (ON CONFLICT on slug). Set createdAt on insert, updatedAt always.
    - `listProjects(query?)`: Select all, optionally filter by host. Return array.
    - `getProject(slug)`: Select by slug, throw NOT_FOUND if missing.

    **`packages/api/src/routes/projects.ts`:**
    - `GET /projects`: Call `listProjects()`. For each project, merge cached scan data (branch, dirty, dirtyFiles, commits). Return `{ projects }`.
    - `GET /projects/:slug`: Validate param with `zValidator('param', projectSlugSchema)`. Call `getProjectWithScanData()`. Return `{ project }`.
    - `POST /projects/refresh`: Trigger `scanAllProjects()` asynchronously (don't await). Return 202 `{ message: 'Scan initiated' }`.
    - All endpoints accept optional `userId` query param (PLAT-02).
    - Export as `projectRoutes`.

    **Update `packages/api/src/app.ts`:**
    Register `.route('/api', projectRoutes)`.

    **Update `packages/api/src/index.ts`:**
    After server starts, load config and run initial scan. Start background poll. On SIGTERM/SIGINT, clear the poll interval before shutting down.

    **Tests:**
    `project-scanner.test.ts`: Unit test the scanner. Create a temporary git repo in a temp directory (use `execSync('git init')` in beforeAll), make a commit, then verify `scanProject()` returns correct branch, dirty status, and commit info. Test cache TTL behavior. Test graceful handling of non-existent paths.

    `projects.test.ts`: Integration test the project routes. Use test app with in-memory database. Seed some project records. Verify GET /projects returns them. Verify GET /projects/:slug returns detail. Verify 404 for unknown slug.
  </action>

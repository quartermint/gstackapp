<objective>
Build the backend enrichment pipeline: extend the captures DB schema with AI and link metadata columns, create the AI categorization service (Vercel AI SDK + structured output), the link metadata extractor (open-graph-scraper), and the enrichment orchestrator that wires them together.

Purpose: This is the "enrich later" half of "persist first, enrich later." Once a capture is saved as "raw," this pipeline categorizes it to a project and extracts link metadata asynchronously.

Output: Three new services (ai-categorizer, link-extractor, enrichment), extended DB schema, and tests validating the pipeline.
</objective>

    **2. Extend Drizzle schema** (`packages/api/src/db/schema.ts`):
    Add these nullable columns to the captures table definition:
    - `aiConfidence: real("ai_confidence")` -- 0-1 float
    - `aiProjectSlug: text("ai_project_slug")` -- slug returned by AI
    - `aiReasoning: text("ai_reasoning")` -- brief AI explanation
    - `linkUrl: text("link_url")` -- first detected URL
    - `linkTitle: text("link_title")` -- OG title
    - `linkDescription: text("link_description")` -- OG description
    - `linkDomain: text("link_domain")` -- hostname
    - `linkImage: text("link_image")` -- OG image URL
    - `enrichedAt: integer("enriched_at", { mode: "timestamp" })` -- when enrichment completed
    Add index: `index("captures_enriched_at_idx").on(table.enrichedAt)`

    **3. Create migration** (`packages/api/drizzle/0002_capture_enrichment.sql`):
    Nine ALTER TABLE statements, each separated by `--> statement-breakpoint` (per established Drizzle migration pattern from 01-02). All columns nullable with no defaults since existing rows won't have this data.

    **4. Update shared schemas** (`packages/shared/src/schemas/capture.ts`):
    - Add the new fields to `captureSchema` (all optional/nullable)
    - Add `aiConfidence`, `aiProjectSlug`, `linkUrl`, `linkTitle`, `linkDescription`, `linkDomain` to `updateCaptureSchema` (all optional)
    - Extend `listCapturesQuerySchema` with optional `stale: z.coerce.boolean().optional()` filter

    **5. Add stale capture query** (`packages/api/src/db/queries/captures.ts`):
    Add `getStaleCaptures(db: DrizzleDb, limit?: number)` function:
    - Query captures WHERE createdAt < (now - 14 days) AND status != 'archived'
    - Order by createdAt ASC (oldest first)
    - Default limit 20
    - Return array of capture objects

    **6. Create AI categorizer** (`packages/api/src/services/ai-categorizer.ts`):
    - Export `categorizeCapture(content: string, projects: Array<{ slug: string; name: string; tagline: string | null }>): Promise<{ projectSlug: string | null; confidence: number; reasoning: string }>`
    - Use `generateText` from `ai` package with `Output.object()` and a Zod schema for structured output
    - Use `openai("gpt-4o-mini")` as default model (configurable via `AI_MODEL` env var)
    - Prompt: provide project list as context, ask AI to match the capture to a project, return slug + confidence + reasoning
    - If AI returns confidence below 0.6, return `{ projectSlug: null, confidence, reasoning }` -- the threshold is applied HERE, not in the caller
    - Export `CONFIDENCE_THRESHOLD = 0.6` as named constant
    - Handle errors gracefully: if AI call fails, return `{ projectSlug: null, confidence: 0, reasoning: "AI categorization failed" }`

    **7. Create link extractor** (`packages/api/src/services/link-extractor.ts`):
    - Export `extractUrls(content: string): string[]` -- regex URL detection (pattern: `/https?:\/\/[^\s<>)"']+/gi`)
    - Export `containsUrl(content: string): boolean`
    - Export `extractLinkMetadata(url: string): Promise<{ title: string | null; description: string | null; domain: string; image: string | null }>`
    - Use `open-graph-scraper` with 5-second timeout and custom user-agent "MissionControl/1.0"
    - On failure: return `{ title: null, description: null, domain: new URL(url).hostname, image: null }` (never lose the URL)

    **8. Create enrichment orchestrator** (`packages/api/src/services/enrichment.ts`):
    - Export `enrichCapture(db: DrizzleDb, captureId: string): Promise<void>`
    - Workflow:
      1. Update status to "pending_enrichment"
      2. Get capture content from DB
      3. Fetch fresh project list from DB (always fresh, never cached -- per research pitfall #2)
      4. Run AI categorization
      5. If content contains URL, extract link metadata for first URL
      6. Update capture with: projectId (from AI if confidence >= threshold, else null), aiConfidence, aiProjectSlug, aiReasoning, linkUrl, linkTitle, linkDescription, linkDomain, linkImage, enrichedAt, status = "enriched"
    - Import `listProjects` from project queries for the project context
    - All DB queries use the DI pattern (db as first param)

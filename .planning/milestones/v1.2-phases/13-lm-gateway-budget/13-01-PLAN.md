**Create `packages/api/src/services/lm-studio.ts`:**

Module-level cache pattern (same as project-scanner background poll). Key implementation details:

```typescript
import type { LmStudioConfig } from "../lib/config.js";

export type LmStudioHealth = "unavailable" | "loading" | "ready";

export interface LmStudioStatus {
  health: LmStudioHealth;
  modelId: string | null;
  lastChecked: Date;
}
```

- `probeLmStudio(config?: LmStudioConfig)`: async function that fetches `${config?.url ?? "http://100.x.x.x:1234"}/v1/models` with AbortController 5-second timeout. Parses response as `{ data?: Array<{ id: string; state?: string }> }`. Finds target model via `model.id.toLowerCase().includes(config?.targetModel ?? "qwen3-coder")`. Determines health: connection error = "unavailable", API up + no target = "loading", target found + state !== "loaded" = "loading", target found + state === "loaded" = "ready". Updates module-level `cachedStatus` and returns it. Catches ALL errors silently and sets "unavailable".

- `getLmStudioStatus()`: returns `cachedStatus` synchronously (zero-latency for routes).

- `startLmStudioProbe(intervalMs = 30_000)`: calls `probeLmStudio()` immediately, then sets up `setInterval`. Returns the interval handle for cleanup. Accepts optional config for URL and target model.

- Default `cachedStatus` initialized as `{ health: "unavailable", modelId: null, lastChecked: new Date() }`.

**Create `packages/api/src/services/budget-service.ts`:**

Pure query functions pattern (follows captures.ts / sessions.ts query pattern). Key implementation details:

```typescript
import { sql } from "drizzle-orm";
import type { DrizzleDb } from "../db/index.js";
import { sessions } from "../db/schema.js";
import type { BudgetThresholds } from "../lib/config.js";
```

- `getWeekStart(now: Date, resetDay: number)`: Walk backward from `now` until `d.getDay() === resetDay`, then set time to 00:00:00.000. Returns Date.

- `getWeeklyBudget(db: DrizzleDb, thresholds: BudgetThresholds, now?: Date)`: Computes `weekStart` via `getWeekStart(now ?? new Date(), thresholds.weekResetDay)`. Queries sessions table: `db.select({ tier: sessions.tier, count: sql<number>\`count(*)\` }).from(sessions).where(sql\`${sessions.startedAt} >= ${Math.floor(weekStart.getTime() / 1000)}\`).groupBy(sessions.tier).all()`. NOTE: `startedAt` is `integer mode:timestamp` storing epoch SECONDS -- must divide JS milliseconds by 1000. Accumulates into `{ opus: 0, sonnet: 0, local: 0, unknown: 0 }`. Computes burnRate: `opus >= hot` = "hot", `opus >= moderate` = "moderate", else "low". Returns `WeeklyBudget` shape with `isEstimated: true` always.

- `suggestTier(taskDescription: string | null, burnRate: BurnRate, localAvailable: boolean)`: Returns null when `burnRate === "low"`. Keyword matching on lowercased description: `TIER_KEYWORDS` map with opus=["architecture","design","complex","plan","security audit"], sonnet=["test","fix","refactor","update","bug","lint","format"], local=["scaffold","boilerplate","template","generate","stub"]. If matched tier is "local" and !localAvailable, skip. If burnRate is "hot" and matched tier is "opus", suggest downgrade to localAvailable ? "local" : "sonnet" with reason. If burnRate is "hot" and no keyword match, suggest sonnet with generic reason. Returns `{ suggestedTier, reason, localAvailable }` or null.

Both services export only pure functions (no side effects except the module-level cache in lm-studio.ts).

    The service follows the discovery-scanner.ts pattern. Key components:

    **1. Imports:**
    ```typescript
    import { execFile as execFileCb } from "node:child_process";
    import { promisify } from "node:util";
    import { createHash } from "node:crypto";
    import pLimit from "p-limit";
    import { getKnowledge, upsertKnowledge } from "../db/queries/knowledge.js";

    import type { DrizzleDb } from "../db/index.js";
    import type Database from "better-sqlite3";
    ```

    **2. Constants:**
    - `SSH_CONNECT_TIMEOUT = 5` (seconds)
    - `SSH_CMD_TIMEOUT = 15_000` (ms, generous for CLAUDE.md reads)
    - `LOCAL_CMD_TIMEOUT = 10_000` (ms)
    - `MAX_FILE_SIZE = 512_000` (500KB guard per pitfall 4)
    - `STALE_AGE_DAYS = 30` (per D-08)
    - `STALE_COMMIT_THRESHOLD = 10` (per D-08)
    - `DEFAULT_INTERVAL_MS = 3_600_000` (1 hour per D-05)

    **3. computeContentHash(content: string): string**
    - Normalize line endings: `content.replace(/\r\n/g, "\n")` (pitfall 2)
    - Return SHA-256 hex digest via `createHash("sha256").update(normalized, "utf-8").digest("hex")`

    **4. readLocalClaudeMd(projectPath: string): Promise<{content, lastModified, commitsSince} | null>**
    - Use `execFile("sh", ["-c", script], { cwd: projectPath, timeout: LOCAL_CMD_TIMEOUT })`
    - Script: `git show HEAD:CLAUDE.md` + delimiter + `git log -1 --format=%aI -- CLAUDE.md` + delimiter + commit count since last CLAUDE.md change
    - Return null on any error (graceful)
    - Skip if content length > MAX_FILE_SIZE (log warning)

    **5. readRemoteClaudeMd(sshHost: string, projectPath: string): Promise<{content, lastModified, commitsSince} | null>**
    - Use `execFile("ssh", ["-o", "ConnectTimeout=5", sshHost, script], { timeout: SSH_CMD_TIMEOUT })`
    - Same script pattern as local but executed over SSH (per D-07: return null on failure, never throw)
    - Skip if content length > MAX_FILE_SIZE

    **6. checkStaleKnowledge(slug, lastModified, commitsSinceUpdate, now?): HealthFindingInput | null**
    - Pure function (per D-08: BOTH thresholds must be met -- AND logic)
    - Returns `{ projectSlug: slug, checkType: "stale_knowledge", severity: "warning", detail: "CLAUDE.md is X days old with Y commits since last update", metadata: { ageDays, commitsSinceUpdate } }` when stale
    - Returns null when fresh

    **7. buildScanTargets(config: MCConfig): Array<{slug, path, host, sshHost?}>**
    - Iterate config.projects
    - For single-host entries: skip `host: "github"` (pitfall 3). Include local and mac-mini entries.
    - For multi-copy entries: prefer local copy over mac-mini (pitfall 6 -- per-slug dedup). If local copy exists in copies array, use it. Otherwise use mac-mini copy.
    - If `config.localSshHost` is set, treat "local" entries as SSH targets (reverse SSH when MC runs on Mac Mini). See open question 3 in research.
    - Return flat array of targets, one per slug.

    **8. scanAllKnowledge(config, db, sqlite): Promise<{scanned, updated, errors}>**
    - Build scan targets via buildScanTargets
    - Use `pLimit(3)` for parallel reads (pitfall 1)
    - For each target:
      - Read CLAUDE.md (local or SSH depending on host)
      - If read returns null: skip (graceful degradation, log warning for SSH failures)
      - Compute content hash
      - Check existing knowledge via `getKnowledge(db, slug)`
      - If hash matches existing: skip DB write (KNOW-02 content-hash caching), but still update lastScannedAt via a lightweight touch
      - If hash differs or no existing: call `upsertKnowledge(sqlite, { projectSlug, content, contentHash, fileSize, lastModified, commitsSinceUpdate })`
      - Check stale knowledge: call `checkStaleKnowledge` and if result, call `upsertHealthFinding`
      - If updated, emit `eventBus.emit("mc:event", { type: "knowledge:updated", id: slug })`
    - After scanning all projects: call `resolveFindings(sqlite, slug, ["stale_knowledge"])` for each project that was scanned successfully but is NOT stale (to clear previous stale findings)
    - Return stats object { scanned, updated, errors }

    **9. startKnowledgeScan(config, db, sqlite, intervalMs?): ReturnType<typeof setInterval>**
    - Follow discovery-scanner pattern exactly: run initial scan, then setInterval
    - Default interval: `intervalMs ?? DEFAULT_INTERVAL_MS` (1 hour per D-05)
    - Log scan stats on completion

    - Mock `execFile` via vi.mock("node:child_process") to control local and SSH output

    - Test stale detection as pure function tests (no mocking needed)
    - Test SSH failure by making mock reject
    - Test multi-copy dedup by configuring multi-copy entry in mock config
    - Test line ending normalization by verifying hash equality for CRLF vs LF content
